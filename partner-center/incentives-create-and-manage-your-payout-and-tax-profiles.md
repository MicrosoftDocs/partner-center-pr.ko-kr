# <a name="create-and-manage-incentives-payout-and-tax-profiles-in-partner-center"></a>파트너 센터 인센티브 지급 및 세금 프로필 만들기 및 관리

**적절한 역할:** 인센티브 관리자 | 계정 관리자 | 전역 관리자

특정 MPN 위치에 대한 인센티브 프로그램의 지급금을 받으려면 유효한 지급 및 세금 프로필을 프로그램 및 MPN 위치와 연결하여 등록을 완료해야 합니다. Microsoft는이 지급 및 세금 프로필을 사용하여 지급금을 제공합니다. 인센티브 프로그램의 규칙에 따라 전자 계좌 이체 또는 결제 신용 전표를 사용할 수 있습니다. 

## <a name="roles-currencies-and-multiple-microsoft-incentive-programs"></a>역할, 통화 및 여러 Microsoft 인센티브 프로그램

지급 및 세금 프로필을 시작하기 전에 아래 정보를 이해하는 것이 중요합니다.

### <a name="roles-and-permissions"></a>역할 및 권한

인센티브 지급을 위해 은행 및 세금 정보를 입력하려면 인센티브 관리자여야 합니다. MPN/계정 관리자인 경우 자신 및/또는 동료를 인센티브 관리자로 할당할 수 있습니다.

인센티브 관리자 권한을 요청해야 하는 경우 MPN 관리자 또는 전역 관리자에게 문의하세요. [파트너 센터 대시보드에](https://partner.microsoft.com/dashboard/)로그인하여 회사에서 이러한 역할이 있는 사람을 확인할 수 있습니다. 오른쪽 위에 있는 **설정** 아이콘에서 **사용자 관리를** 선택한 다음 전역 관리자를 필터링합니다.

인센티브 사용자는 인센티브 소득 및 지불 세부 정보 및 보고서를 볼 수 있지만 은행 및 세금 세부 정보는 편집할 수 없습니다.

### <a name="choose-your-disbursement-currency"></a>지출 통화 선택

인센티브 결제는 결제 프로필을 설정할 때 선택한 통화로 이루어집니다. 지불액은 Microsoft에서 매월 설정한 환율을 사용하여 계산됩니다. 선택한 통화로 인해 값을 변경해야 합니다.

### <a name="using-different-profiles-for-different-microsoft-programs"></a>다른 Microsoft 프로그램에 다른 프로필 사용

회사가 여러 인센티브 프로그램에 등록된 경우 모든 프로그램에 대해 동일한 결제 계정을 사용하거나 다른 프로그램에 다른 결제 계정을 사용하도록 선택할 수 있습니다.

## <a name="create-and-manage-payout-and-tax-profiles-in-partner-center"></a>파트너 센터에서 지급 및 세금 프로필 만들기 및 관리

아래 섹션에서는 파트너 센터 결제 및 세금 프로필을 만들고 관리하는 과정을 안내합니다.

>[!IMPORTANT]
>파트너 센터 결제 및 세금 프로필을 만들거나 관리하려면 인센티브 관리자여야 합니다. 인센티브 역할은 각 인센티브 프로그램에서 각 MPN 위치에 할당되어야 합니다. 파트너 센터에서 동기 관리자를 추가 하는 방법에 대 한 자세한 내용은 [사용자 계정 만들기](create-user-accounts-and-set-permissions.md)를 참조 하세요.

## <a name="access-the-payout-and-tax-section-in-partner-center"></a>파트너 센터의 지급 및 세금 섹션 액세스

1. Azure Active Directory (Azure AD) 계정 (회사 계정)을 사용 하 여 [파트너 센터 대시보드에](https://partner.microsoft.com/dashboard/) 로그인 하거나, 할당 된 경우 적절 한 전자 메일 주소를 사용 하 여 로그인 합니다.

   - 하나의 Azure AD 계정 내에 여러 도메인을 등록할 수 있습니다. 전역 관리자에 게 문의 하 여 연결 된 도메인을 확인 합니다.
   - 도메인에만 로그인 할 수 @onmicrosoft.com 있고 추가 도메인이 필요한 경우 계정 관리자에 게 문의 하 여 AZURE AD 계정에 도메인을 추가 합니다.
   - **회사 또는 학교** 계정 또는 **개인 계정을** 선택 하 라는 메시지가 표시 되 면 **회사 또는 학교 계정** 을 선택 합니다.

2. 기어 아이콘을 선택 하 여 **설정** 메뉴를 열고 **계정 설정** 을 선택 합니다.

3. **계정 설정** 메뉴에서 **지급 및 세금** 을 선택 합니다.

## <a name="assign-payout-and-tax-profiles-to-individual-programs"></a>개별 프로그램에 지급 및 세금 프로필 할당

1. [파트너 센터 대시보드에](https://partner.microsoft.com/dashboard/)로그인 하 고 기어 아이콘을 선택 하 여 **설정** 메뉴를 엽니다. 

2. **계정 설정** 을 선택 하 고 **지급 및 세금 섹션** 을 확장 한 다음 **지급 및 세금 프로필 할당** 을 선택 합니다. 
   
   프로그램 목록이 표시 됩니다. 프로그램 옆의 화살표를 선택 하 여 프로필 세부 정보를 확인 합니다. 

3. **세금 프로필** 드롭다운 메뉴에서 원하는 세금 프로필을 선택 하거나 새 프로필을 만드는 옵션을 선택 합니다. 새 프로필을 만드는 옵션을 선택 하면 적절 하 게 리디렉션됩니다.  팝업 창에서 **계속** 을 선택 합니다. 새 세금 프로필을 만드는 프로세스는 다음과 같습니다.

4. **지불 방법** 을 선택 합니다.

   - 지불 방법으로 **전자 은행 전송** 을 선택한 경우 원하는 지불 프로필을 선택 하거나 새 프로필을 만드는 옵션을 선택 합니다. 새 프로필을 만드는 옵션을 선택 하면 적절 하 게 리디렉션됩니다. 팝업 창에서 계속을 선택 합니다. 새 지불 프로필을 만드는 프로세스는 아래에 제공 되어 있습니다.

   - **신용 정보** 를 지불 방법으로 선택한 경우 확인을 완료 합니다. 이렇게 하면 참조 된 SAP 번호가 조직에 속해 있음을 확인 합니다.

    >[!NOTE]
    >Microsoft 비즈니스 엔터티가 여러 개 있는 경우 각 엔터티에 대해 지불 프로필을 선택 해야 합니다.

    >[!NOTE]
    >지불 방법의 가용성은 동기 프로그램의 규칙에 따라 달라 집니다.

    - 특정 동기 프로그램에 대해 로컬 Microsoft 자회사에서 location MPN ID를 지불 하 고 LRD (제한 된 위험 배포자) 신용 메모를 지불 방법으로 허용 하는 경우 지불 프로필은 LRD 신용 메모 지불 방법으로 미리 채워집니다. 각 동기 프로그램 및 위치 MPN ID에 대 한 LRD 신용 메모 지불 방법 행에서 지불 프로필 섹션의 **상태로 확인 또는** **확인 필요** 가 표시 됩니다.
    
       신용 메모 지불을 받으려면 location MPN and 결제 방법과 연결 된 CSP 테 넌 트 ID 세부 정보를 확인 하 고 확인 하는 데 **필요한 확인** 을 선택 합니다. **신용 메모 세부 정보** 대화 상자에서 제공 된 CSP 테 넌 트 ID 및 세부 정보가 올바른지 확인 하 고 확인 합니다. 둘 이상의 테 넌 트 ID가 표시 되는 경우 지불액을 받으려는 CSP 테 넌 트 ID를 신중 하 게 선택 합니다. 그런 다음, **확인** 을 선택 하 여 회사 세부 정보가 올바른지 확인 하 고, 선택한 CSP 테 넌 트 ID에 대해 동기를 지불 해야 한다는 것을 승인 합니다.
 
      상태가 **확인** 됨으로 표시 되 면 CSP 테 넌 트 ID의 할당이 완료 된 것 이며 추가 작업이 필요 하지 않습니다. 확인을 선택 하 여 할당에 대 한 세부 정보를 볼 수 있습니다.
   
      파트너가 면세 적용을 명시적으로 요청해야 하는 국가에서는 인센티브 프로그램 및 위치 MPN의 세금 프로필 섹션에 있는 세금 프로필 옆에 세금 예외를 적용하는 옵션이 있습니다. 이 확인란을 선택하면 인센티브 크레딧 메모에 면세 혜택이 적용됩니다. 
   
      현재 LRD 크레딧 메모 결제 방법은 Microsoft 상거래 인센티브 프로그램에 대해 오스트레일리아, 뉴질랜드 및 캐나다 파트너에만 사용할 수 있습니다. MCI 프로그램에 등록된 이러한 3개 국가에서 직접 청구 파트너 또는 간접 공급자이고 사용 가능한 결제 방법으로 LRD 크레딧 메모가 표시되지 않는 경우 테넌트 ID가 관련 파트너 MPN 위치 계정과 연결되어 있는지 확인합니다. 이에 대한 자세한 내용은 [조직 프로필을 업데이트하는 방법을 읽어보세요.](update-your-partner-profile.md)

    
5. **통화** 를 선택합니다.

6. 모든 결제 필드를 완료했으면 **제출을** 선택합니다.

## <a name="set-up-a-default-bank-profile"></a>기본 은행 프로필 설정

기본 은행 프로필을 설정하고 MPN 위치에 할당할 수 있습니다. 이러한 기본 프로필은 해당 MPN 위치에 대한 후속 등록에 Microsoft에서 사용됩니다. 

1. [파트너 센터 대시보드에](https://partner.microsoft.com/dashboard/)로그인한 다음, 기어 아이콘을 선택하여 **설정**   메뉴를 엽니다. 

2.  **계정 설정을** 선택하고 **지급 및 세금** 섹션을 확장한 다음 지급 및 세금 프로필을 **선택합니다.** 

3. 결제 프로필 섹션에서 기본 **프로필** **관리를** 선택합니다. 

4. 기본 은행 프로필을 만들려면 **기본 은행 프로필 추가를** 선택합니다. 

5. 회사의 사용 가능한 은행 프로필 목록에서 은행 프로필을 선택하고 이 은행 프로필에 사용할 통화를 선택한 다음, 이 기본 프로필을 적용할 MPN 위치 목록을 선택합니다.

6. 선택을 완료한 후 **완료를** 선택합니다. 완료 단추는 모든 필수 필드가 완료될 때까지 클릭할 수 없습니다. 

>[!NOTE]
>동일한 은행 및 통화 페어링을 여러 위치에 적용할 수 있습니다. MPN 위치에 기본 프로필과 통화 조합이 한 번 할당 된 경우 나중에 기본 프로필 할당을 위해 위치 드롭다운에서 더 이상 표시 되지 않습니다. 기본 선택 항목이 삭제 되 면 이후 기본 프로필 할당을 위해 MPN 위치가 다시 나타납니다. 각 은행 프로필 및 통화 조합은 편집 가능한 고유 행으로 추가 됩니다.

7. 필요한 변경 내용이 모두 추가 되 면 **저장** 을 선택 합니다.  

## <a name="create-your-bank-profile"></a>은행 프로필 만들기

은행 프로필은 회사 수준에서 만들어집니다. 이를 통해 하나의 은행 프로필을 여러 MPN ID에 할당 하 고 회사 내의 프로그램을 프로 파일링 할 수 있습니다. 서로 다른 은행 및 세금 규칙이 적용 될 수 있으므로 은행 프로필을 다른 국가에 적용 하는 경우 예외가 발생할 수 있습니다.

>[!NOTE]
>다음 페이지에서는 별표가 있는 필드가 필요 합니다. 필드가 무엇 인지 모르는 경우 정보 아이콘을 선택 합니다. 

1. **세부 정보** 페이지에서 다음 필드를 완료 합니다. **프로필 이름:** 이 지불 프로필을 식별 하는 고유한 이름을 입력 합니다.
    **은행 계좌 위치:** 회사의 은행이 있는 국가입니다.
    **지불 방법:** 파트너 센터에 대 한 기본 결제 방법은 전자 은행 전송입니다.

2. **다음** 을 선택합니다.

3. **은행 계좌** 페이지에서 사용자 정보를 입력 합니다. 이 페이지에 표시 되는 필드는 국가별로 다를 수 있습니다. 

4. **다음** 을 선택합니다.

5. **수혜자** 페이지에서 적절 한 정보를 입력 합니다. 수혜자는 회사에서 사용자 계정에 대해 논의 해야 하는 경우 은행에 연락 하는 사람입니다.

6. 필드가 완료 되 면 **마침** 을 선택 하 고 **확인** 을 선택 하 여 은행 프로필을 만듭니다.

**지급 및 세금 프로필** 페이지로 리디렉션됩니다. 새 프로필의 상태는 유효성 검사가 완료 될 때까지 **보류 중인 Microsoft 유효성 검사** 를 반영 합니다. 이 프로세스는 최대 48시간이 걸릴 수 있습니다. 유효성 검사가 완료되면 프로필 상태에 승인 또는 **작업 필요** **가** 반영됩니다. **작업이 필요한** 경우 위의 단계를 반복하여 필요한 정보를 제공합니다. 

## <a name="create-your-tax-profile"></a>세금 프로필 만들기

다음 절차에 따라 조직에 필요한 세금 정보를 Microsoft에 제공합니다. 이 섹션의 페이지는 동적이며 국가 또는 지역에 따라 달라집니다. 올바른 세금 정보를 식별하는 데 도움이 필요한 경우 해당 국가의 적절한 정부 출처에 문의하세요.

미국 내 파트너 회사의 경우 W8 또는 W9 양식을 작성하기 위한 정보가 필요한 경우 다음 주소를 통해 IRS 사이트로 안내합니다.

- [http://www.irs.gov/pub/irs-pdf/iw8.pdf](http://www.irs.gov/pub/irs-pdf/iw8.pdf)
- [http://www.irs.gov/pub/irs-pdf/iw9.pdf](http://www.irs.gov/pub/irs-pdf/iw9.pdf)

>[!IMPORTANT]
> 회사에 대한 세부 정보만 입력합니다. 개인 정보를 입력하지 않습니다.

1. 비즈니스 **프로필** 페이지에서 필수 필드를 완료한 후 **다음을** 선택합니다. 

2. **설치** 페이지에서 회사에 적용되는 옵션을 선택합니다.

   - 회사가 미국만 통합된 경우 또는 개인용 프로필인 경우 왼쪽의 옵션을 선택합니다.
   - 회사가 미국 외부에 통합된 경우 오른쪽의 옵션을 선택한 다음 목록에서 해당 국가/지역을 선택합니다.

3. **다음** 을 선택합니다. 

4. 세금 **상태** 페이지에서 필요한 정보를 입력한 후 **다음을** 선택합니다. 이 페이지의 필드는 국가별로 다릅니다. 세부 정보를 입력합니다. 

5. 추가 **설명서** 페이지에서 필수 필드를 선택하고 **다음을** 선택합니다. 

6. **찾아보기를** 선택하여 국가 또는 지역에 필요한 문서를 업로드합니다. 문서 이름이 표시되면 **업로드를** 선택합니다. 

7. 문서를 제거 해야 하는 경우 **제거** 를 선택 합니다.

8. 저장 하 고 계속 하려면 **마침** 을 선택 합니다.

9. 팝업 메시지에서 **확인** 을 선택 합니다. **지급 및 세금 설치** 페이지로 돌아갑니다.
 
## <a name="update-expired-tax-profiles"></a>만료 된 세금 프로필 업데이트

1. [파트너 센터 대시보드에](https://partner.microsoft.com/dashboard/)로그인 하 고 기어 아이콘을 선택 하 여 **설정** 메뉴를 엽니다.

1. **계정 설정** 을 선택 하 고 **지급 및 세금** 섹션을 확장 한 다음 **지급 및 세금 프로필** 을 선택 합니다.

1. **세금 프로필** 을 선택 합니다.

1. 만료 **날짜** 열을 확인 하 고 만료 되었거나 곧 만료 될 세금 프로필로 이동 합니다.

1. **편집** 을 선택합니다.

1. 세금 양식 섹션에서 새 세부 정보를 제공 하 여 세금 양식을 업데이트 합니다.
