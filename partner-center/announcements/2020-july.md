## <a name="changes-to-enterprise-voice-and-audio-conferencing-extended-dial-out-launches"></a><a name="10"></a>Enterprise Voice 및 오디오 회의 확장 전화 접속 시작에 대한 변경 내용
### <a name="categories"></a>범주

- 날짜: 2020-07-31
- 제품/시장

### <a name="impacted-audience"></a>영향을 받는 대상

직접 공급자, 간접 공급자 및 간접 재판매인

### <a name="summary"></a>요약

Enterprise Voice와 E5 및 A5 with Calling Minutes 출시가 취소되었습니다. 미국 및 캐나다에 대한 확장 전화 접속은 2020년 9월 1일로 연기되었습니다.

### <a name="details"></a>세부 정보

Enterprise Voice 플랜 1 및 2와 E5 및 A5 플랜에 포함되어 함께 제공되는 국내 통화 시간은 이전에 계획된 날짜인 2020년 8 월 1일에 출시되지 않습니다. 출시가 무기한 보류되었습니다. 

미국 및 캐나다에 대한 확장 전화 접속의 출시는 2020년 9월 1일로 연기되었습니다.

#### <a name="enterprise-voice-plans-and-e5-and-a5-with-calling-minutes-launch-canceled"></a>Enterprise Voice 플랜과 E5 및 A5 with Calling Minutes 출시 취소
 
**질문과 대답**

- **Enterprise Voice와 E5/A5 with Calling Minutes를 출시하지 않는 이유는 무엇인가요?** 급격하게 발전하는 시장 상황으로 인해 출시하지 않기로 했습니다. 
- **나중에 Enterprise Voice 및 E5/A5 with Calling Minutes를 출시할 예정인가요?** 지금은 알 수 없습니다. 시장 및 판매 데이터를 계속 평가하여 출시 일정을 다시 잡을 것인지 결정할 것입니다. 
- **이 변경 내용이 모든 채널에 적용되나요?** 예, 이전에 계획된 어떤 채널(VL, CSP 및 웹 다이렉트)에서도 출시하지 않을 것입니다. 
- **SKU가 8월 VL 가격표에 계속 표시되나요?** 아니요. 영향을 받는 SKU는 가격표 미리 보기에 표시되지만, 8월 1일에 게시되는 가격표에서는 제거될 예정입니다. 미리 보기 가격표를 다운로드한 현장 및 파트너는 해당 복사본을 8월 1일에 게시되는 업데이트된 가격표로 바꿔야 합니다. 
- **제품이 8월 CSP 가격표에 표시되나요?** 아니요. 영향을 받는 SKU는 가격표 미리 보기에 표시되지만, 8월 1일에 게시되는 CSP 가격표에서는 제거될 예정입니다. 미리 보기 가격표를 다운로드한 현장 및 파트너는 해당 복사본을 8월 1일에 게시되는 업데이트된 가격표로 바꿔야 합니다.
- **발표된 전화 시스템, 오디오 회의 또는 고급 통신 도입 제품에도 영향이 있나요?** 아니요. 이러한 제품은 계획대로 계속 제공됩니다. 
- **웹 다이렉트 구매를 위한 고객 포털에 제품이 표시되나요?** 아니요.

## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>파트너 센터 API에 대한 변경 내용

### <a name="categories"></a>범주

- 날짜: 2020-07-28
- 기능

### <a name="impacted-audience"></a>영향을 받는 대상

파트너 센터 API를 사용하는 CSP 프로그램에 참여하는 Direct Bill 파트너 및 간접 공급자 

### <a name="summary"></a>요약

파트너 센터 팀에서 공개된 파트너 센터 API를 평가했습니다. 이러한 API 중 일부는 유효하지 않으며 더 이상 지원되지 않습니다. 2020년 8월 31일에 사용이 중지됩니다.

### <a name="details"></a>세부 정보

CSP 파트너 센터 팀에서 2020년 8월 31일에 사용 중지될 몇 가지 API를 확인했습니다. 이러한 API는 몇 년 전에 설계된 인터페이스를 나타내며, 대부분이 더 이상 작동하지 않습니다. 팀에서는 2020년 마지막 분기의 다음 SDK 릴리스에서 이러한 엔드포인트에 대한 참조를 제거하고 있습니다. 서비스 해제할 API는 다음과 같습니다. 파트너는 앞으로 이러한 URI를 참조하지 않도록 해야 합니다.

**파트너 서비스 요청** - 이러한 엔드포인트를 통해 파트너가 지원 항목 문자열을 가져온 다음, 파트너 서비스 요청을 프로그래밍 방식으로 제출할 수 있도록 설정했습니다. 시간이 지남에 따라 이 API는 불안정하게 되며 지원되지 않습니다. 파트너는 파트너 지원 티켓을 만들기 위해 파트너 센터 사용자 인터페이스에 연결됩니다. 사용자 환경은 파트너에게 추가 정보를 제공하면서 문제가 있는 영역에 추천되는 단계 및 문서와 같은 지원 사례를 만듭니다. 또한 파트너 센터 팀은 문제 영역과 관련된 정보를 캡처하도록 서비스 요청 양식을 사용자 지정하여 파트너 티켓 만들기 환경을 향상시켰습니다. 온라인 양식으로 수집된 정보는 엔지니어가 문제를 더 빠르고 정확하게 해결하는 데 도움이 됩니다. 이러한 URI는 참조되거나 사용되지 않아야 하며, 2020년 8월 말에 서비스가 해제됩니다. 이러한 변경 내용은 지원 항목 가져오기에만 영향을 주고, **파트너** 서비스 요청 메서드를 만듭니다. 다른 모든 지원 관련 API는 예상대로 계속 작동합니다.

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Office 서비스 인시던트** - 이러한 엔드포인트는 파트너가 파트너의 모든 서비스에서 서비스 인시던트/중단에 대한 집계 보기를 가져올 수 있도록 설계되었습니다. 그러나 이러한 엔드포인트에서 반환하는 데이터는 부실하고 유용하지 않습니다. 이러한 엔드포인트는 7월 말까지 서비스를 해제할 예정입니다. 파트너 센터 팀은 앞으로 '모든 고객' 서비스 인시던트 및 상태 시나리오를 재평가하고 이 영역에서 가치를 추가할 수 있는 기회를 찾습니다. 이러한 URI에 대한 참조를 제거해야 합니다.

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

파트너는 [Office 365 서비스 통신 API](https://docs.microsoft.com/office/office-365-management-api/office-365-service-communications-api-reference)를 사용하여 고객 서비스에 대한 최신 상태를 가져올 수 있습니다.

**TopProduct** - 이 엔드포인트는 일반 공급에 적용되지 않는 개념 증명으로 만들어졌습니다. 이는 파트너가 언젠가 파트너 센터 카탈로그에서 렌더링하는 상위 제품을 관리할 수 있도록 설계되었습니다. 그러나 이 API는 개발이 끝나지 않은 미완성 상태이며, 2020년 7월 31일까지만 사용됩니다. 파트너는 이에 대한 모든 참조를 제거해야 합니다.

- {base-URL}/v1/preferences/catalog/products

### <a name="next-steps"></a>다음 단계

- 파트너는 해당 솔루션에서 위의 API를 더 이상 참조하지 않도록 해야 합니다.
- 클라우드 솔루션 공급자 파트너는 추가 설명이 필요한 경우 [파트너 센터 SDK 및 API Yammer 커뮤니티](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560)를 방문하여 질문할 수 있습니다.

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>.NET Standard의 SDK 릴리스(v1.16.1-preview)

### <a name="categories"></a>범주

- 날짜: 2020-07-21
- 기능

### <a name="impacted-audience"></a>영향을 받는 대상

파트너 센터 .NET SDK를 사용하는 CSP 프로그램에 참여하는 Direct Bill 파트너 및 간접 공급자 

### <a name="summary"></a>요약

2020년 7월 21일부터 파트너는 업데이트된 퍼블릭 **파트너 센터 SDK GitHub 샘플**과 함께 .NET SDK v1.16.1-preview( **.NET Standard SDK Microsoft 파트너 센터**의 미리 보기 버전)의 다운로드를 시작할 수 있습니다.

### <a name="details"></a>세부 정보

CSP 프로그램에 참여하는 Direct Bill 파트너 및 간접 공급자는 최신 미리 보기 버전의 .NET Standard SDK를 다운로드할 수 있습니다. Microsoft 파트너 센터 SDK를 .NET Framework에서 .NET Standard 2.0 플랫폼으로 마이그레이션할 예정입니다. 이번 변화로 인해 SDK가 .NET Framework 4.6.1 이상을 사용하는 기존 애플리케이션과 호환됩니다. .NET Standard는 .NET Core 2.0 이상을 지원합니다.

.NET Standard에서 SDK를 요청하는 파트너의 요구가 매우 많았습니다. 현재 버전의 SDK는 파트너가 ASP.NET Framework로 작성된 애플리케이션을 강제로 사용하도록 합니다. 이 경우 .NET Core 애플리케이션에 비해 속도가 느리고 비용이 많이 듭니다. 따라서 .NET Standard로 마이그레이션하면 성능 또는 .NET Core와의 호환성 문제를 해결하는 데 도움이 됩니다.
 
**중요 사항**
- Microsoft의 .NET Framework 4.8용 릴리스는 마지막 릴리스이므로 더 이상 릴리스되지 않습니다.
- 대부분의 라이브러리가 현재 프레임워크 버전을 지원하지 않을 수 있는 .NET Core 또는 .NET Standard용으로 릴리스되므로 향후 개발이 제한됩니다.
- .NET Core는 간단한 특성으로 인해 컨테이너에 사용하는 것이 좋습니다.
- Windows Server 및 Linux용 고성능 서버 런타임은 .NET Core를 최고 성능의 웹 프레임워크로 만듭니다.

### <a name="next-steps"></a>다음 단계

- 최신 버전의 [.NET SDK v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview) 다운로드
- 버전: .NET SDK v1.16.1-preview 
- 샘플 코드: [샘플 코드](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- 미리 보기 사용 가능한 날짜: 2020년 7월 21일 
- GA: 미리 보기로부터 3주 후(파트너가 마이그레이션하는 데 충분한 시간을 제공하기 위해)

- 상태: 자격이 있는 간접 공급자 및 해당 간접 재판매인이 사용할 수 있습니다. 2021년 1월에 일반적으로 사용 가능
 
 
### <a name="impacted-audience"></a>영향을 받는 대상

CSP 간접 재판매인 및 자격이 있는 CSP 간접 공급자 

### <a name="details"></a>세부 정보

Microsoft는 모든 조직의 디지털 변환 과정을 지원하기 위해 최선을 다하고 있습니다. 이는 단순히 기술에만 투자하는 것이 아니라 기술 획득에서 지원 환경에 이르기까지 엔드투엔드 고객 및 파트너의 경험을 검토함으로써 이를 수행합니다. 이 작업을 통해 상거래 플랫폼에 투자하여 자체적으로 혁신할 수 있는 기회를 확인했습니다.  또한 새로운 상거래 경험을 발전시켜 파트너에게 더 많은 기회를 제공하는 방법을 개선할 수 있도록 라이선싱 프로그램을 단순화하고 있습니다. 

고객과 파트너에게 더 나은 구매 환경을 제공하기 위해 상거래 기능을 개선하면서, CSP(클라우드 솔루션 공급자) 프로그램 내에 영구 소프트웨어 라이선스의 가용성을 포함하여 오퍼 포트폴리오를 확장하고 있습니다. CSP 내에 소프트웨어 라이선스를 추가하면 참여하는 파트너가 고객에게 다양한 솔루션 및 관리 서비스를 제공하는 데 집중할 수 있습니다. 또한 고객의 클라우드 전환을 강화하고 파트너에게 하이브리드 클라우드 환경에 필요한 유연성을 제공합니다.   

7월 1일부터 CSP 프로그램의 고객과 함께 온-프레미스 소프트웨어에 대한 라이선스를 거래할 수 있는 일련의 자격이 있는 간접 공급자와 해당 간접 재판매인을 통해 이 기능을 강화합니다. 자격이 있는 파트너는 Microsoft에서 관리하며, 현재 CSP 내 소프트웨어 판매가 지원되는 지역에서 작동하는 오픈 라이선스 프로그램에 대해 활성 CSP 간접 공급자 및 Microsoft 공인 배포자가 모두 있어야 합니다. 2021년 1월부터 Microsoft는 이 기능을 추가로 확장하고 모든 CSP 직접 청구 파트너, 간접 공급자 및 간접 재판매인을 통해 CSP 프로그램 내에서 소프트웨어 라이선스를 판매합니다.

### <a name="summary"></a>요약

FY20 CSP 인센티브 프로그램은 2020년 9월 30일까지 연장되었습니다. FY21 프로그램은 2020년 10월 1일에 시작됩니다.   

### <a name="impacted-audience"></a>영향을 받는 대상

CSP 직접 청구 파트너, 간접 공급자 및 간접 재판매인

### <a name="details"></a>세부 정보

원래 2020년 6월 30일까지 예정된 FY20 CSP 인센티브 프로그램은 2020년 9월 30일까지 연장됩니다. 변경 내용을 반영하도록 프로그램 가이드가 업데이트되었습니다. 현재의 요금은 2020년 9월 30일까지 계속됩니다. CSP 인센티브 프로그램 제품 추록 파일은 2020년 7월 1일부터 적용되도록 업데이트됩니다. 이러한 파일은 관련 [MPN의 CSP 인센티브 프로그램 자산 컬렉션](https://aka.ms/partnerincentives)에서 찾을 수 있습니다.

FY21 CSP 인센티브 프로그램은 2020년 10월 1일에 시작됩니다. FY21 CSP 인센티브 프로그램에 대한 자세한 내용은 2020년 8월 1일에 공유됩니다.
