# <a name="how-indirect-providers-can-help-their-indirect-resellers-accept-the-microsoft-partner-agreement-mpa"></a>간접 공급자가 간접 대리점이 MPA(Microsoft 파트너 계약)를 수락하는 데 도움이 되는 방법

**적절한 역할:** 전역 관리자 | 관리 에이전트 | 판매 에이전트

다음 질문과 답변은 간접 대리점이 MPA를 수락하는 방법과 이유를 이해하는 데 도움이 되는 잠재적인 문제 및 해결 방법을 다룹니다. 

클라우드 솔루션 공급자 프로그램의 간접 공급자, 직접 청구 파트너 및 간접 대리점은 이 정보가 유용하다는 것을 알게 됩니다.

## <a name="questions-on-mpa-program-and-policy"></a>MPA 프로그램 및 정책에 대한 질문

#### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-accept-the-mpa"></a>간접 대리점이 MPA를 수락하지 않는 경우 법적 의미는 무엇인가요?
 
**답변** 본인(간접 대리점)이 MPA를 수락하지 않는 한 CSP 프로그램에서 간접 공급자(배포자)와 거래할 수 없습니다. 비즈니스를 수행하려는 각 [CSP 지역에](regional-authorization-overview.md) 대해 MPA를 수락해야 합니다. 간접 대리점이 해당 CSP 지역의 모든 간접 공급자와 CSP 트랜잭션을 수행하려면 CSP 지역당 하나 이상의 MPA 서명이 필요합니다. 

#### <a name="is-mpa-available-in-local-languages"></a>MPA를 로컬 언어로 사용할 수 있나요?
 
**답변** 아니요. 영어로만 사용할 수 있습니다. 

#### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-accept-the-mpa-why-is-that"></a>제 재판매인은 이미 간접 재판매인으로 등록되어 있다고 생각합니다. 이러한 재판매인은 클라우드 구독을 재판매할 수 있었지만 MPA를 수락할 수 있는 옵션은 없습니다. 그 이유는 무엇인가요?

**답변** 대리점이 파트너 센터 로그인할 수 있지만 MPA를 수락하거나 CSP Indirect Reseller 프로그램에 등록되었는지 확인하는 옵션이 표시되지 않는 경우 MPN 프로그램에만 등록되었지만 간접 Reseller로 CSP에 등록되지 않았을 가능성이 큽니다. 과거에는 CSP 프로그램에 공식적으로 등록하지 않은 경우에도 대리점이 간접 공급자를 통해 거래할 수 있도록 허용했습니다.

이제 공식 CSP 간접 대리점이 될 차례입니다.
이는 CSP에서 거래한 "Advisor" 파트너에도 적용됩니다. 간접 대리점을 사용 하려면 간접 대리점의 모든 CSP 지역에 활성 CSP 간접 재판매인 계정이 필요 합니다.

#### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-accept-the-mpa-in-each-csp-country"></a>다양 한 국가에서 여러 개의 배포자 (간접 공급자)를 사용 합니다. 각 CSP 국가에서 MPA를 수락 해야 하나요?


**답변** MPN ID에 대해 CSP 지역 당 하나 이상의 MPA 서명이 필요 합니다. 예를 들어 프랑스에 MPN ID가 있고, 프랑스 및 수락 된 MPA에서 CSP 간접 재판매인 프로그램에 등록 한 경우 전체 [CSP 유럽 지역](regional-authorization-overview.md#europe-region-and-market) 에서 규격을 준수 하 고 다른 유럽 국가의 MPA를 수락 하지 않아도 됩니다. 전체 CSP 유럽 지역에서 모든 간접 공급자를 사용할 수 있습니다. 

#### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-accept-two-mpas-one-as-direct-bill-and-one-as-indirect-reseller-"></a>프랑스에서 직접 청구 파트너 이며 이탈리아에서 간접 공급자가 사용 되 고 있습니다. 두 개의 MPAs를 직접 청구 하 고 간접 재판매인으로 사용 해야 하나요?

**답변**  CSP 지역에서 직접 청구 MPA를 이미 수락한 경우 CSP 간접 재판매인 MPA를 수락할 필요가 없습니다. 그러나 MPN 계정이 파트너 센터에 있고 간접 공급자와 공유 하는 MPN ID가 파트너 센터에 있는지 확인 해야 합니다. 

#### <a name="do-we-have-to-keep-accepting-the-indirect-reseller-mpa-agreement-every-year"></a>매년 간접 재판매인 MPA 계약을 계속 수락 해야 하나요?


**답변** 매년 동의할 필요가 없습니다. 간접 공급자를 사용 하려면 준수를 유지 하기 위해 CSP 지역 당 하나 이상의 MPA를 수락 해야 합니다.


#### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-accept-the-agreement"></a>구독이 변경 되지 않는 경우 재판매인이 규약에 동의 하는 경우에는 문제가 있나요? 

**답변** 간접 대리점은 MPA를 수락 하지 않는 경우 간접 공급자와 거래에서 차단 됩니다. Microsoft의 성과급이 영향을 받습니다. 

#### <a name="will-i-be-off-boarded-from-partner-center-if-i-dont-accept-the-mpa-what-are-the-exact-risks-if-resellers-dont-accept-the-mpa"></a>MPA에 동의 하지 않는 경우 파트너 센터에서 등록을 해제 하 시겠습니까? 대리점에서 MPA를 수락 하지 않는 경우 정확한 위험은 무엇입니까?

**답변** 간접 대리점은 거래에서의 배포자 (간접 공급자)로 차단 됩니다. 가까운 장래에 간접 재판매인의 보드를 해제할 계획이 없습니다. 

#### <a name="my-reseller-says-that-the-mpa-was-already-accepted-by-an-employee-who-left-the-company-do-they-need-to-accept-again"></a>대리점은 이미 회사를 떠난 직원 들이 MPA를 수락 했다는 것을 의미 합니다. 다시 수락 해야 하나요?

**답변** 아니요

#### <a name="my-reseller-recently-changed-company-name-but-accepted-the-mpa-already-do-they-need-to-accept-again"></a>리셀러가 최근에 회사 이름을 변경 했지만 이미 MPA를 수락 했습니다. 다시 수락 해야 하나요?

**답변** 아니요

#### <a name="do-we-have-to-communicate-with-our-resellers-about-accepting-the-mpa-or-microsoft-will-work-with-resellers-directly-to-accept--mpa"></a>MPA를 수락 하는 것과 관련 하 여 대리점과 의견을 교환 해야 하나요? 아니면 Microsoft에서 대리점을 직접 사용 하 여 MPA를 수락 합니까?
 
**답변** 간접 공급자는 모든 간접 대리점과 명확 하 게 통신 하 고 MPA를 수락 하지 않는 영향을 인식 하는 것은 사용자의 책임입니다. 또한 MPA 수락에 적절 한 지원 및 지침을 제공 해야 합니다. Solvable 되지 않는 중요 한 문제에 대해서는이 파트너 센터 지원 토픽을 사용 하 여 Microsoft로 리디렉션해야 합니다.

#### <a name="are-there-any-documents-that-describe-how-resellers-can-accept-on-the-mpa-and-confirm-if-the-latest-mpa-was-accepted"></a>대리점에서 MPA를 수락할 수 있는 방법을 설명 하 고 최신 MPA가 수락 되었는지 확인 하는 문서가 있나요? 

**답변** 다음은 간접 재판매인이 MPA를 수락 하 고 규정을 준수 하는 데 도움이 되는 몇 가지 유용한 공개 링크입니다.

**답변** MPN 계정을 사용 하 여 Microsoft와의 멤버 자격을 관리 해야 합니다. 사용자가 만든 자회사 수에 따라 하나의 MPN (Global MPN ID) 및 하나 이상의 PLA (Location IDs)를 제공 합니다. CSP 계정은 사용자의 위치 MPN Id와 연결 되며 간접 공급자를 대상으로 하거나 성과급을 관리 하는 데 사용 됩니다.

**답변** MPN 글로벌 계정은 본사와 동일하며 MPN 위치는 자회사입니다.  고유한 전역 MPN ID(PGA MPN ID)와 하나 또는 여러 개의 고유한 위치 MPN ID(PLA MPN ID)가 있습니다. MPN 계정으로 파트너 센터 로그인하고 **설정,** 계정 **설정,** **식별자** 페이지로 이동하여 MPN ID를 확인할 수 있습니다. 

다른 국가에 자회사가 있는 경우 해당 국가에서 MPN 위치를 만들고 위치 MPN ID를 얻을 수 있습니다. 그런 다음 이러한 위치 MPN ID를 사용하여 CSP 프로그램에 등록하고, 간접 공급자와 정보를 공유하고, 인센티브가 부여됩니다. 

•[파트너 계정의 위치 관리](manage-locations.md)

문제가 발생하는 경우 파트너 센터 내에서 [지원 티켓을](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=1af7f3a0-1757-3543-4b6a-c945c3ad187b) 제기합니다. 


#### <a name="i-dont-know-which-mpn-id-to-use-while-enrolling-into-csp-indirect-reseller-program"></a>CSP Indirect Reseller 프로그램에 등록하는 동안 사용할 MPN ID를 모릅니다.

**답변** [CSP Indirect Reseller 프로그램에 등록하기](indirect-reseller-tasks-in-partner-center.md)전에 파트너 센터 활성 MPN 계정이 있어야 합니다. MPN ID 국가 및 CSP 계정 국가를 성공적으로 연결하려면 일치해야 하며 MPN ID는 파트너 센터 있어야 합니다. MPN 계정에 로그인합니다(MPN 및 CSP 로그인 자격 증명이 다를 수 있음). 설정   ->  계정 설정 조직 프로필 식별자로 이동하여  ->    ->   CSP 계정의 국가에 맞는 "위치" **형식의** MPN ID를 찾습니다. CSP 등록 프로세스 중에 MPN ID를 CSP 계정에 연결하기 전에 [MPN 위치에](manage-locations.md) 대해 자세히 알아봅니다.

#### <a name="how-do-i-check-which-mpn-id-is-associated-with-my-csp-account"></a>내 CSP 계정과 연결된 MPN ID를 확인할 어떻게 할까요? 있나요?

**답변** CSP Indirect Reseller 계정에 로그인합니다. **설정** -> **계정 설정** -> **조직 프로필** -> **식별자** 로 이동합니다. 페이지의 CSP 영역에서 **연결된 MPN ID는** CSP 계정과 연결된 MPN ID이며 CSP 트랜잭션에 대한 간접 공급자에게 제공되어야 합니다. 

#### <a name="how-can-i-view-all-my-mpn-ids"></a>내 MPN ID를 모두 보려면 어떻게 해야 합니까?

**답변** MPN 계정 자격 증명을 사용 하 여 파트너 센터에 로그인 합니다 (MPN 및 CSP 로그인 자격 증명은 다를 수 있음). **설정** 에서 **계정 설정**  ->  **조직 프로필**  ->  **식별자** 를 선택 하 여 MPN id를 확인 합니다. 새 MPN location/자회사를 추가 하려면 **법적 프로필** 로 이동 하 여 **위치 추가** 를 클릭 합니다.

## <a name="questions-about-csp-regional-authorization"></a>CSP 지역 권한 부여에 대 한 질문

#### <a name="what-is-a-csp-region"></a>CSP 지역은 무엇 인가요?

**답변** CSP 지역은 CSP 비즈니스를 수행할 수 있는 하나 이상의 국가로 구성 됩니다. 비즈니스에는 간접 공급자 작업, 고객과의 연결, 거래 및 발생 성과급이 포함 됩니다. CSP 지역에서 업무를 수행 하려면 CSP 계정 (office 365 회사 계정)이 하나 이상 필요 합니다. [CSP 지역](regional-authorization-overview.md)에 대해 자세히 알아보세요.

CSP 간접 재판매인 인 경우 동일한 지역의 간접 공급자와 파트너를 사용 해야 하며, 동일한 지역의 고객 에게만 판매할 수 있습니다.

#### <a name="why-cant-i-associate-my-mpn-id-to-my-tenant"></a>MPN ID를 내 테 넌 트에 연결할 수 없는 이유는 무엇입니까?

**답변**  MPN ID를 테 넌 트에 연결할 수 없고 오류가 표시 되는 경우 MPN ID와 테 넌 트가 동일한 국가에 있지 않음을 의미 합니다. 연결이 작동 하려면 둘 다 동일한 국가에 있어야 합니다. 이 요구 사항은 Microsoft의 성과급이 국가 및 MPN ID 수준에 따라 지불 되기 때문입니다. 예를 들어 프랑스에 CSP 계정이 있는 경우 유료 성과급을 얻기 위해 프랑스에도 MPN location (MPN ID)이 필요 합니다. 영국의 MPN ID 는 동일한 CSP 지역에 있지만 작동 하지 않습니다. 

이 문제를 해결 하려면 CSP country와 동일한 국가에서 MPN 위치를 만듭니다. [MPN 위치를 관리](manage-locations.md)하는 방법에 대해 자세히 알아보세요.

#### <a name="i-have-a-csp-indirect-reseller-account-in-france-can-i-partner-with-indirect-provider-in-germany"></a>프랑스에 CSP 간접 재판매인 계정이 있습니다. 독일 간접 공급자와 파트너 관계를 맺을 수 있나요?

**답변** 예. CSP Indirect Reseller 계정이 CSP 지역에서 활성화되면 동일한 계정을 사용하여 전체 지역에서 비즈니스를 수행할 수 있습니다. [CSP 지역에](regional-authorization-overview.md)대해 자세히 알아보세요.

#### <a name="i-am-an-indirect-reseller-in-the-us-can-i-partner-with-an-indirect-provider-in-uk"></a>저는 미국의 간접 대리점입니다. 영국 간접 공급자와 파트너 관계를 맺을 수 있나요?

**답변** 아니요. CSP는 지역 프로그램입니다. 간접 대리점 및 간접 공급자의 CSP 계정은 비즈니스를 수행하려면 동일한 CSP 지역에 있어야 합니다. 이 문제를 해결하려면 유럽 지역에서 새 CSP 계정을 만들고 CSP 계정 국가와 동일한 국가에서 MPN 위치를 만든 다음 간접 공급자의 관계 요청을 수락해야 합니다. 예를 들어, 프랑스에 CSP Indirect Reseller 계정을 만들고, 프랑스에 MPN 위치를 만든 다음, 프랑스 또는 CSP 유럽 지역의 다른 국가에 있는 간접 공급자의 초대를 수락할 수 있습니다.

#### <a name="can-i-have-more-than-one-mpn-and-csp-accounts-in-partner-center"></a>파트너 센터 MPN 및 CSP 계정을 두 개 이상 사용할 수 있나요?

**답변** 회사에서 수행하는 비즈니스의 CSP 지역에 따라 MPN 계정(로그인)과 하나 또는 여러 CSP 계정(로그인)만 가질 수 있습니다. 예를 들어 미국(자회사), 미국(자회사) 및 오스트레일리아(자회사)의 위치가 있는 미국(HQ)에 MPN 계정이 있을 수 있습니다.
MPN 계정과 동일한 로그인 자격 증명을 가진 하나의 CSP 계정(Office 365 작업 계정)이 미국에 있을 수 있습니다. 두 번째 CSP 계정은 미국에 있을 수 있습니다. 유럽 지역에서 CSP 비즈니스를 관리하기 위한 로그인 자격 증명이 서로 다르고, 오스트레일리아에서 CSP 비즈니스를 관리하기 위한 다른 두 개의 로그인 자격 증명이 있는 오스트레일리아의 세 번째 로그인 자격 증명이 있습니다.

세 가지 CSP 계정은 모두 MPN ID 위치 계정과 연결되어야 비즈니스를 수행하며 해당 자회사에서 인센티브가 획득됩니다. MPN 역량 및 혜택은 MPN 계정에서만 관리될 수 있습니다.

#### <a name="i-am-a-csp-indirect-reseller-in-us-and-want-to-expand-my-business-to-europe-what-do-i-need-to-do-in-partner-center"></a>저는 미국에서 CSP Indirect Reseller 비즈니스를 유럽으로 확장하려고 합니다. 파트너 센터 어떻게 해야 합니까?

**답변** 비즈니스를 수행하려는 국가에 MPN 위치가 필요하며, 동일한 국가의 CSP Indirect Reseller 프로그램에 등록해야 합니다. 새 [MPN 위치 및 CSP 등록을](manage-locations.md#prerequisites-in-order-to-add-a-new-account-for-a-csp-business)만드는 방법에 대해 자세히 알아봅니다.

## <a name="question-about-migrating-your-mpn-account-from-pmc-to-partner-center"></a>PMC에서 파트너 센터 MPN 계정 마이그레이션에 대한 질문

#### <a name="i-want-to-migrate-my-mpn-account-from-partner-membership-center-to-partner-center-how-can-i-do-it-i-do-not-know-who-was-my-mpn-admin-in-pmc-how-can-i-migrate"></a>내 MPN 계정을 Partner Membership Center 파트너 센터 마이그레이션하려고 합니다. 어떻게 할 수 있나요? PMC의 MPN 관리자가 누구인지 모릅니다. 어떻게 마이그레이션할 수 있나요? 

**답변** [MPN을 PMC에서 PC로 마이그레이션하는](https://assetsprod.microsoft.com/mpn/migrate-pmc-pc-mpa-guide.pptx)방법에 대한 단계별 가이드를 다운로드합니다.

MPN 계정을 마이그레이션한 후 MPA에 서명하는 방법에 대해 자세히 알아보려면 [microsoft-partner-agreement를](microsoft-partner-agreement.md#download-the-step-by-step-guide-thats-right-for-where-you-are-in-the-mpa-signing-process)참조하세요.

문제가 있는 경우 파트너 센터 내에서 [지원 티켓을](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=2da4e1c7-3b4b-5672-6227-56bc75a936b4) 제기합니다.

#### <a name="how-do-i-check-if-my-mpn-id-is-migrated-into-partner-center"></a>내 MPN ID가 파트너 센터 마이그레이션되는지 어떻게 할까요? 확인하세요.

**답변**  모든 역할(MPN 및 CSP 로그인 자격 증명이 다를 수 있음)을 사용하여 MPN 계정에 로그인합니다. 

**계정 설정의** **파트너** 탭에서 **법적 프로필** 페이지를 선택합니다. 프로그램 정보 아래에 "MPN 프로그램 상태"가 표시되면 MPN ID가 파트너 센터. "파트너" 탭에 "MPN 프로그램 상태"가 표시되지 않으면 CSP 계정으로 로그인했거나 MPN 계정이 아직 파트너 센터 않습니다. MPN 계정을 PMC에서 PC로 마이그레이션해야 할 수도 있습니다.

## <a name="questions-about-mpn-and-csp-account-verification"></a>MPN 및 CSP 계정 확인에 대한 질문

#### <a name="i-enrolled-in-mpn-program-and-as-a-csp-indirect-reseller-program-but-my-legal-business-profile-verification-status-is-in-rejected-state-what-should-i-do"></a>MPN 프로그램 및 CSP 간접 대리점 프로그램에 등록했지만 법적 비즈니스 프로필 확인 상태가 거부됨 상태입니다.   어떻게 해야 합니까?

- [조직 프로필 확인](update-your-partner-profile.md)

직접 문제를 해결할 수 없는 경우 CSP 계정으로 로그인하여 파트너 센터 내에서 지원 티켓을 제기합니다.

**답변**  CSP 간접 대리점 법적 비즈니스 프로필이 영업일 기준 5일이 지난 후 **자격 보류 중** 상태인 경우 파트너 센터 지원을 클릭하여 파트너 센터 내에서 [지원](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=345795c0-26bd-dd95-d291-b78ed4a8edce)티켓을 제기합니다.

## <a name="questions-about-working-in-partner-center"></a>파트너 센터 작업에 대한 질문

#### <a name="i-dont-know-who-the-partner-center-admin-is-who-can-accept-mpa"></a>MPA를 수락할 수 있는 파트너 센터 관리자가 누구인지 모릅니다.

**답변** 회사의 전역 관리자는 MPA를 수락해야 합니다. 회사에서 전역 관리자 역할이 할당된 사용자를 확인하려면 **사용자 관리로 이동합니다.** 검색 상자에 전역 관리자를 입력합니다. 전역 관리자 역할이 할당된 사람의 이름을 가져옵니다. 이러한 사용자는 MPA를 수락할 수 있습니다. 회사에 전역 관리자가 없는 경우 회사에 전역 관리자가 [필요한 경우 수행할 작업을 참조하세요.](no-mpn-admin.md)

:::image type="content" source="images/user-management-clean.png" alt-text="전역 관리자를 찾는 방법":::


#### <a name="my-reseller-says-that-in-partner-center-they-dont-see-an-option-to-accept-the-mpa-how-can-i-best-support-them"></a>내 대리점은 파트너 센터 MPA를 수락하는 옵션이 표시되지 않는다고 말합니다. 어떻게 가장 잘 지원할 수 있나요?

**답변**  대리점이 파트너 센터 로그인할 수 있지만 MPA를 수락하는 옵션이 표시되지 않는 경우:

1. 왼쪽의 CSP 탭 아래에 "개요" 페이지가 표시하나요? 

:::image type="content" source="images/csp.png" alt-text="파트너 센터 메뉴에서 CSP를 찾을 수 있는 위치":::

파트너 센터 메뉴의 CSP 아래에 "개요"가 표시 되지 않으면 CSP 프로그램에 아직 등록 하지 않은 것입니다. 재판매인은 [csp 프로그램의 간접 공급자 작업에](indirect-reseller-tasks-in-partner-center.md)설명 된 대로 Csp 간접 재판매인으로 등록 하는 프로세스를 시작 해야 합니다. 등록을 완료 한 후에는 MPA를 수락할 수 있습니다.

2. 개요가 표시 되 면 **설정**  ->  **계정 설정**  ->  **법률**  ->  **파트너** 또는 **재판매인** 탭으로 이동 하 여 프로그램 정보로 스크롤합니다. 간접 재판매인 상태 옆에 표시 되는 상태는 무엇 인가요?


상태가 다른 것 이지만 **활성** 상태인 경우에는 법적 비즈니스 프로필 확인이 아직 진행 중이거나 거부 되었음을 의미 합니다. 동일한 페이지에서 재판매인은 확인 상태를 확인 하 고 **자세한 정보** 를 클릭 하 여 문제를 해결 해야 합니다 .이 문제를 해결 하려면 재판매인은 확인을 진행 하는 데 필요한 설명서를 제공할 수 있습니다.

:::image type="content" source="images/active-status.png" alt-text="상태가 활성입니다.":::

재판매인이 법적 비즈니스 프로필 확인을 통과 하도록 지원 해야 하는 경우 다음 옵션을 선택 하 여 티켓을 열도록 요청 합니다.

- 파트너 센터 메뉴에서 **지원** 을 선택한 다음 **일반** > 계정 확인 >을 선택 하 고에 대 한 도움이 필요한 단계를 선택 합니다.


#### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-accept-mpa"></a>파트너 센터에서 MPA를 수락 하는 데 적합 한 역할이 있는지 확인 어떻게 할까요??

**답변** 전역 관리자만 MPA를 수락할 수 있습니다. 파트너 센터에서 역할을 확인 하려면 다음을 수행 합니다.

1. 파트너 센터에서 CSP 간접 재판매인 계정에 로그인 하 고 먼저 CSP 간접 재판매인 계정에 등록 했는지 확인 합니다. 

2. **사용자 관리** 로 이동 하 여 로그인 한 계정에 "전역 관리자" 역할이 할당 되어 있는지 확인 합니다. 그렇지 않으면 MPA를 허용 하는 데 사용할 수 없습니다. 사용자 관리 페이지의 검색 상자에서 전역 관리자를 검색 하 고 사용자에 게 MPA를 수락 하도록 요청 합니다.

:::image type="content" source="images/user-management-clean.png" alt-text="파트너 센터에서 사용자의 역할 식별":::


#### <a name="how-do-i-check-if-my-company-has-accepted-the-mpa"></a>회사에서 MPA를 수락 했는지 확인 어떻게 할까요??

**답변** CSP 간접 재판매인 계정으로 파트너 센터에 로그인 합니다.
CSP의 관리 에이전트 또는 판매 에이전트의 경우 규약 페이지로 이동 하 여 Microsoft 파트너 계약이 표시 되는지 확인할 수 있습니다. 

:::image type="content" source="images/agreement-new.png" alt-text="계정 설정에서 계약 찾기":::


전역 관리자 인 경우 CSP 개요 페이지로 이동 합니다. 녹색 배너가 표시 되는 경우 모두 해당 CSP 지역에서 간접 공급자를 사용 하 여 모든 설정 됩니다. 그렇지 않은 경우 **수락 및 계속** 단추가 MPA를 수락 하 고 CSP 계정을 활성화 하는 것을 볼 수 있습니다.

:::image type="content" source="images/banner-boxed.png" alt-text="녹색 배너":::

#### <a name="i-want-to-accept-mpa-but-i-dont-see-any-place-button-to-accept-terms-to-do-so"></a>MPA를 수락 하려고 하지만 어떤 장소 (약관에 동의 하는 단추)가 표시 되지 않나요?

**답변** CSP 간접 재판매인 계정 자격 증명을 사용 하 여 파트너 센터에 로그인 합니다. CSP 개요 페이지로 이동합니다.  **수락 및 계속** 단추가 표시 되지 않고, mpa가 수락 되었음을 알리는 녹색 배너가 없으면 mpa를 수락 하는 데 적합 한 역할이 없음을 의미 합니다. MPA를 수락 하려면 전역 관리자에 게 문의 해야 합니다.

:::image type="content" source="images/agreement-two.png" alt-text="규약 찾기":::
