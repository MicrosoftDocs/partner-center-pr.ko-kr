# <a name="how-indirect-providers-can-help-their-indirect-resellers-accept-the-microsoft-partner-agreement-mpa"></a>간접 공급자가 간접 대리점이 MPA(Microsoft 파트너 계약)를 수락하는 데 도움이 되는 방법

**적절한 역할:** 전역 관리자 | 관리 에이전트 | 판매 에이전트

다음 질문과 답변은 간접 대리점이 MPA를 수락하는 방법과 이유를 이해하는 데 도움이 되는 잠재적인 문제 및 해결 방법을 다룹니다. 

클라우드 솔루션 공급자 프로그램의 간접 공급자, 직접 청구 파트너 및 간접 대리점은 이 정보가 유용하다는 것을 알게 됩니다.

## <a name="questions-on-mpa-program-and-policy"></a>MPA 프로그램 및 정책에 대한 질문

#### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-accept-the-mpa"></a>간접 대리점이 MPA를 수락하지 않는 경우 법적 의미는 무엇인가요?
 
**답변** 본인(간접 대리점)이 MPA를 수락하지 않는 한 CSP 프로그램에서 간접 공급자(배포자)와 거래할 수 없습니다. 비즈니스를 수행하려는 각 [CSP 지역에](regional-authorization-overview.md) 대해 MPA를 수락해야 합니다. 간접 대리점이 해당 CSP 지역의 간접 공급자와 CSP 트랜잭션을 수행하려면 CSP 지역당 하나 이상의 MPA 서명이 필요합니다. 

#### <a name="is-mpa-available-in-local-languages"></a>MPA를 로컬 언어로 사용할 수 있나요?
 
**답변** 아니요. 영어로만 사용할 수 있습니다. 

#### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-accept-the-mpa-why-is-that"></a>제 재판매인은 이미 간접 재판매인으로 등록되어 있다고 생각합니다. 이러한 재판매인은 클라우드 구독을 재판매할 수 있었지만 MPA를 수락할 수 있는 옵션은 없습니다. 왜 그럴까요?

**답변** 대리점이 파트너 센터 로그인할 수 있지만 MPA를 수락하거나 CSP Indirect Reseller 프로그램에 등록되었는지 확인하는 옵션이 표시되지 않는 경우 MPN 프로그램에만 등록되었지만 간접 Reseller로 CSP에 등록되지 않았을 가능성이 큽니다. 과거에는 CSP 프로그램에 공식적으로 등록되지 않은 경우에도 대리점이 간접 공급자를 통해 거래할 수 있도록 허용했습니다.

이제 공식 CSP 간접 대리점이 될 차례입니다.
이는 CSP에서 거래한 "Advisor" 파트너에도 적용됩니다. 간접 대리점이 간접 공급자와 거래하려면 Microsoft는 모든 CSP 비즈니스 지역에 활성 CSP Indirect Reseller 계정이 필요합니다.

#### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-accept-the-mpa-in-each-csp-country"></a>여러 국가에서 여러 간접 공급자(배포자)와 협력합니다. 각 CSP 국가에서 MPA를 수락해야 합니까?


**답변** MPN ID에 대해 CSP 지역당 하나 이상의 MPA 서명이 필요합니다. 예를 들어 프랑스에 MPN ID가 있고 프랑스 CSP Indirect Reseller 프로그램에 등록하고 MPA를 수락한 경우 전체 [CSP 유럽 지역에서](regional-authorization-overview.md#europe-region-and-market) 규정을 준수하며 다른 유럽 국가에서 MPA를 수락할 필요가 없습니다. 전체 CSP 유럽 지역에서 간접 공급자와 거래할 수 있습니다. 

#### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-accept-two-mpas-one-as-direct-bill-and-one-as-indirect-reseller-"></a>저는 프랑스에 있는 직접 청구 파트너이며 이탈리아에 있는 간접 공급자와 거래하는데, 하나는 직접 청구이고 다른 하나는 간접 대리점인 두 개의 MPA를 수락해야 하나요?

**답변**  CSP 지역에서 직접 청구 MPA를 이미 수락한 경우 CSP 간접 대리점 MPA를 수락할 필요가 없습니다. 그러나 MPN 계정이 파트너 센터 있고 간접 공급자와 공유하는 MPN ID가 파트너 센터 있는지 확인해야 합니다. 

#### <a name="do-we-have-to-keep-accepting-the-indirect-reseller-mpa-agreement-every-year"></a>매년 간접 대리점 MPA 계약에 계속 동의해야 합니까?


**답변** 매년 수락할 필요가 없습니다. 간접 공급자와 거래하려면 규정 준수를 유지하려면 CSP 지역당 하나 이상의 MPA를 수락해야 합니다.


#### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-accept-the-agreement"></a>구독이 변경되지 않는 경우 대리점이 계약에 동의하는지 여부가 중요합니까? 

**답변** 간접 대리점이 MPA를 수락하지 않으면 간접 공급자와의 거래가 차단됩니다. Microsoft 인센티브가 영향을 받게 됩니다. 

#### <a name="will-i-be-off-boarded-from-partner-center-if-i-dont-accept-the-mpa-what-are-the-exact-risks-if-resellers-dont-accept-the-mpa"></a>MPA를 수락하지 않으면 파트너 센터 끄나요? 대리점이 MPA를 수락하지 않는 경우 정확한 위험은 무엇인가요?

**답변** 간접 대리점은 간접 공급자(배포자)와의 거래가 차단됩니다. 가까운 시일 안에 간접 대리점에 오프보치할 계획은 없습니다. 

#### <a name="my-reseller-says-that-the-mpa-was-already-accepted-by-an-employee-who-left-the-company-do-they-need-to-accept-again"></a>제 대리점에서 퇴사한 직원이 MPA를 이미 수락했다고 말했습니다. 다시 동의해야 합니까?

**답변** 아니요

#### <a name="my-reseller-recently-changed-company-name-but-accepted-the-mpa-already-do-they-need-to-accept-again"></a>내 대리점이 최근에 회사 이름을 변경했지만 MPA를 이미 수락했습니다. 다시 동의해야 합니까?

**답변** 아니요

#### <a name="do-we-have-to-communicate-with-our-resellers-about-accepting-the-mpa-or-microsoft-will-work-with-resellers-directly-to-accept--mpa"></a>MPA 수락에 대해 대리점과 통신해야 합니까? 아니면 Microsoft에서 직접 대리점과 협력하여 MPA를 수락하나요?
 
**답변** 간접 공급자는 모든 간접 대리점과 명확하게 통신하고 MPA를 수락하지 않는 것에 대한 영향을 인식하도록 해야 합니다. 또한 MPA를 수락할 때 적절한 지원 및 지침을 제공해야 합니다. 해결할 수 없는 중요한 문제의 경우 이 파트너 센터 지원 항목을 사용하여 Microsoft로 리디렉션해야 합니다.

#### <a name="are-there-any-documents-that-describe-how-resellers-can-accept-on-the-mpa-and-confirm-if-the-latest-mpa-was-accepted"></a>대리점이 MPA에 동의하고 최신 MPA가 수락되었는지 확인하는 방법을 설명하는 문서가 있나요? 

**답변** 다음은 간접 대리점이 MPA를 수락하고 규정을 준수하는 데 도움이 되는 몇 가지 유용한 공개 링크입니다. 

1.  [CSP용 MPA(Microsoft 파트너 계약)(MPN](microsoft-partner-agreement.md) 계정을 마이그레이션하고 파트너 센터 MPA를 수락하는 단계별 지침 포함)
2.  [CSP 프로그램에서 간접 공급자와 협력](indirect-reseller-tasks-in-partner-center.md)
3.  [파트너 계정의 위치 관리](manage-locations.md)

## <a name="questions-about-your-mpn-and-csp-account-structure"></a>MPN 및 CSP 계정 구조에 대한 질문

#### <a name="what-is-the-difference-between-my-mpn-and-csp-accounts-and-how-are-they-related"></a>내 MPN 계정과 CSP 계정 간의 차이점은 무엇이며 어떻게 관련되어 있나요?

**답변** MPN 계정을 사용하여 Microsoft의 멤버 자격을 관리해야 합니다. 만드는 자회사 수에 따라 하나의 글로벌 MPN ID(PGA) 및 하나 또는 여러 위치 MPN ID(PLA)를 제공합니다. CSP 계정은 위치 MPN ID와 연결되며 간접 공급자와 거래하거나 인센티브 관리에 사용됩니다.

- [Microsoft 파트너 네트워크 멤버십 혜택](mpn-overview.md)


위치 MPN ID와 연결된 CSP 계정을 사용하면 간접 공급자와 작업하고, 고객을 관리하고, 인센티브도 받을 수 있습니다.
    
- [CSP 파트너를 위한 MPA(Microsoft 파트너 계약)](microsoft-partner-agreement.md)

#### <a name="what-is-the-difference-between-global-mpn-id-and-location-mpn-id"></a>글로벌 MPN ID와 위치 MPN ID의 차이점은 무엇인가요?

**답변** MPN 글로벌 계정은 본사와 동일하며 MPN 위치는 자회사입니다.  고유한 전역 MPN ID(PGA MPN ID)와 하나 또는 여러 개의 고유한 위치 MPN ID(PLA MPN ID)가 있습니다. MPN 계정으로 파트너 센터 로그인하고 **설정,** 계정 **설정,** **식별자** 페이지로 이동하여 MPN ID를 확인할 수 있습니다. 

다른 국가에 자회사가 있는 경우 해당 국가에서 MPN 위치를 만들고 위치 MPN ID를 얻을 수 있습니다. 그런 다음 이러한 위치 MPN ID를 사용하여 CSP 프로그램에 등록하고, 간접 공급자와 정보를 공유하고, 인센티브가 부여됩니다. 

•[파트너 계정의 위치 관리](manage-locations.md)

문제가 발생하는 경우 파트너 센터 내에서 [지원 티켓을](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=1af7f3a0-1757-3543-4b6a-c945c3ad187b) 제기합니다. 


#### <a name="i-dont-know-which-mpn-id-to-use-while-enrolling-into-csp-indirect-reseller-program"></a>CSP Indirect Reseller 프로그램에 등록하는 동안 사용할 MPN ID를 모릅니다.

**답변** [CSP Indirect Reseller 프로그램에 등록하기](indirect-reseller-tasks-in-partner-center.md)전에 파트너 센터 활성 MPN 계정이 있어야 합니다. MPN ID 국가 및 CSP 계정 국가를 성공적으로 연결하려면 일치해야 하며 MPN ID는 파트너 센터 있어야 합니다. MPN 계정에 로그인합니다(MPN 및 CSP 로그인 자격 증명이 다를 수 있음). 설정   ->  계정 설정 조직 프로필 식별자로 이동하여  ->    ->   CSP 계정의 국가에 일치하는 "위치" **유형으로** MPN ID를 찾습니다. CSP 등록 프로세스 중에 MPN ID를 CSP 계정에 연결하기 전에 [MPN 위치에](manage-locations.md) 대해 자세히 알아봅니다.

#### <a name="how-do-i-check-which-mpn-id-is-associated-with-my-csp-account"></a>내 CSP 계정과 연결된 MPN ID를 확인할 어떻게 할까요? 있나요?

**답변** CSP Indirect Reseller 계정에 로그인합니다. **설정** -> **계정 설정** -> **조직 프로필** -> **식별자** 로 이동합니다. 페이지의 CSP 영역에서 **연결된 MPN ID는** CSP 계정과 연결된 MPN ID이며 CSP 트랜잭션에 대한 간접 공급자에게 제공되어야 합니다. 

#### <a name="how-can-i-view-all-my-mpn-ids"></a>내 MPN ID를 모두 보려면 어떻게 해야 합니까?

**답변** MPN 계정 자격 증명을 사용하여 파트너 센터 로그인합니다(MPN 및 CSP 로그인 자격 증명이 다를 수 있음). **설정에서** **계정 설정** 조직  ->  **프로필**  ->  **식별자를** 선택하여 MPN ID를 봅니다. 새 MPN 위치/자회사를 추가하려면 **법적 프로필로** 이동하여 **위치 추가를** 클릭합니다.

## <a name="questions-about-csp-regional-authorization"></a>CSP 지역 권한 부여에 대한 질문

#### <a name="what-is-a-csp-region"></a>CSP 지역이란?

**답변** CSP 지역은 CSP 비즈니스를 수행할 수 있는 하나 이상의 국가로 구성됩니다. 예를 들어 비즈니스에는 간접 공급자와의 작업, 고객과의 연결, 거래 및 인센티브 발생이 포함됩니다. CSP 지역에서 비즈니스를 수행하려면 하나 이상의 CSP 계정(office 365 작업 계정)이 필요합니다. [CSP 지역에](regional-authorization-overview.md)대해 자세히 알아보세요.

CSP Indirect Reseller 경우 동일한 지역의 간접 공급자와 파트너 관계를 맺어야 하며 동일한 지역의 고객에게만 판매할 수 있습니다.

#### <a name="why-cant-i-associate-my-mpn-id-to-my-tenant"></a>내 MPN ID를 내 테넌트와 연결할 수 없는 이유는 무엇인가요?

**답변**  MPN ID를 테넌트와 연결할 수 없고 오류가 표시되는 경우 MPN ID와 테넌트는 동일한 국가에 있지 않음을 의미합니다. 연결이 작동하려면 둘 다 동일한 국가에 있어야 합니다. 이 요구 사항은 Microsoft 인센티브가 국가 및 MPN ID 수준별로 지급되기 때문입니다. 예를 들어 프랑스에 CSP 계정이 있는 경우 유료 인센티브 받기 위해 프랑스에도 MPN 위치(MPN ID)가 필요합니다. MPN ID(미국) 는 동일한 CSP 지역에 있지만 작동 하지 않습니다. 

이 문제를 해결 하려면 CSP country와 동일한 국가에서 MPN 위치를 만듭니다. [MPN 위치를 관리](manage-locations.md)하는 방법에 대해 자세히 알아보세요.

#### <a name="i-have-a-csp-indirect-reseller-account-in-france-can-i-partner-with-indirect-provider-in-germany"></a>프랑스에 CSP 간접 재판매인 계정이 있습니다. 독일에서 간접 공급자와 협력 하 여 연결할 수 있나요?

**답변** 예로. Csp 간접 재판매인 계정이 CSP 지역에서 활성 상태 이면 동일한 계정을 사용 하 여 전체 지역에서 비즈니스를 수행할 수 있습니다. [CSP 지역](regional-authorization-overview.md)에 대해 자세히 알아보세요.

#### <a name="i-am-an-indirect-reseller-in-the-us-can-i-partner-with-an-indirect-provider-in-uk"></a>미국에서 간접적인 재판매인입니다. UK에서 간접 공급자와 파트너 시킬 수 있나요?

**답변** 아니요. CSP는 지역 프로그램입니다. 비즈니스를 수행 하려면 간접 재판매인 및 간접 공급자의 CSP 계정이 동일한 CSP 지역에 있어야 합니다. 이 문제를 해결 하려면 유럽 지역에서 새 CSP 계정을 만들고, CSP 계정 국가와 동일한 국가에서 MPN 위치를 만든 다음, 간접 공급자의 관계 요청을 수락 해야 합니다. 예를 들어 프랑스에서 CSP 간접 재판매인 계정을 만들고 프랑스에서 MPN 위치를 만든 다음, 프랑스 또는 CSP 유럽 지역의 다른 국가 중 하나에 있는 간접 공급자의 초대를 수락할 수 있습니다.

#### <a name="can-i-have-more-than-one-mpn-and-csp-accounts-in-partner-center"></a>파트너 센터에 둘 이상의 MPN 및 CSP 계정을 사용할 수 있나요?

**답변** 회사에서 수행 하 고 있는 비즈니스의 CSP 영역에 따라 하나의 MPN 계정 (로그인) 및 하나 이상의 CSP 계정 (로그인)을 사용할 수 있습니다. 예를 들어 미국 (계열사), 영국 (자회사) 및 오스트레일리아 (자회사)의 위치를 사용 하 여 미국 (본부)에서 MPN 계정을 사용할 수 있습니다.
MPN 계정과 동일한 로그인 자격 증명을 포함 하는 하나의 CSP 계정 (Office 365 회사 계정)을 미국에 포함할 수 있습니다. 영국의 두 번째 CSP 계정을 사용할 수 있습니다. 유럽 지역에서 CSP 비즈니스를 관리 하는 데 다른 로그인 자격 증명이 있고 다른 두 개의 로그인 자격 증명을 포함 하는 오스트레일리아의 세 번째 자격 증명을 통해 오스트레일리아의 CSP 비즈니스를 관리할 수 있습니다.

비즈니스를 수행 하 고 해당 자회사에서 성과급을 획득 하려면 3 개의 CSP 계정을 모두 MPN ID 위치 계정과 연결 해야 합니다. MPN 역량 및 혜택은 MPN 계정 으로만 관리할 수 있습니다.

#### <a name="i-am-a-csp-indirect-reseller-in-us-and-want-to-expand-my-business-to-europe-what-do-i-need-to-do-in-partner-center"></a>저는 미국에서 CSP 간접 재판매인 이며 내 비즈니스를 유럽으로 확장 하려고 합니다. 파트너 센터에서 수행 해야 하는 작업은 무엇 인가요?

**답변** 회사에서 업무를 수행 하려는 국가에 MPN 위치가 필요 하며 동일한 국가의 CSP 간접 재판매인 프로그램에 등록 해야 합니다. 새 [MPN 위치 및 CSP 등록](manage-locations.md#prerequisites-in-order-to-add-a-new-account-for-a-csp-business)만들기에 대해 자세히 알아보세요.

## <a name="question-about-migrating-your-mpn-account-from-pmc-to-partner-center"></a>PMC에서 파트너 센터로 MPN 계정을 마이그레이션하는 방법에 대 한 질문

#### <a name="i-want-to-migrate-my-mpn-account-from-partner-membership-center-to-partner-center-how-can-i-do-it-i-do-not-know-who-was-my-mpn-admin-in-pmc-how-can-i-migrate"></a>파트너 구성원 센터에서 파트너 센터로 MPN 계정을 마이그레이션하려고 합니다. 어떻게 할 수 있나요? PMC에서 내 MPN 관리자가 누구 인지 알 수 없습니다. 어떻게 마이그레이션할 수 있나요? 

**답변** [MPN를 PMC에서 PC로 마이그레이션하](https://assetsprod.microsoft.com/mpn/migrate-pmc-pc-mpa-guide.pptx)는 방법에 대 한 단계별 가이드를 다운로드 합니다.

MPN 계정을 마이그레이션한 후에는 MPA에 서명 하는 방법에 대해 자세히 알아보고 [microsoft 파트너 계약](microsoft-partner-agreement.md#download-the-step-by-step-guide-thats-right-for-where-you-are-in-the-mpa-signing-process)을 읽습니다.

문제가 발생 하는 경우 파트너 센터 내에서 [지원 티켓](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=2da4e1c7-3b4b-5672-6227-56bc75a936b4) 을 생성 합니다.

#### <a name="how-do-i-check-if-my-mpn-id-is-migrated-into-partner-center"></a>MPN ID가 파트너 센터로 마이그레이션 되었는지 확인 어떻게 할까요??

**답변**  모든 역할을 사용 하 여 MPN 계정에 로그인 합니다 (MPN 및 CSP 로그인 자격 증명은 다를 수 있음). 

**계정 설정** 에서 **파트너** 탭의 **법적 프로필** 페이지를 선택 합니다. 프로그램 정보 아래에 "MPN Program status"가 표시 되는 경우 MPN ID는 파트너 센터에 있습니다. "파트너" 탭에 "MPN Program status"가 표시 되지 않으면 CSP 계정으로 로그인 했거나 MPN 계정이 파트너 센터에 아직 없는 것입니다. MPN 계정을 PMC에서 PC로 마이그레이션해야 할 수 있습니다.


:::image type="content" source="images/legal-info-mpn.png" alt-text="MPN 프로그램 정보는 어디에서 볼 수 있습니다.":::

## <a name="questions-about-mpn-and-csp-account-verification"></a>MPN 및 CSP 계정 확인에 대 한 질문

#### <a name="i-enrolled-in-mpn-program-and-as-a-csp-indirect-reseller-program-but-my-legal-business-profile-verification-status-is-in-rejected-state-what-should-i-do"></a>MPN 프로그램 및 CSP 간접 재판매인 프로그램으로 등록 되었지만 법적 비즈니스 프로필 확인 상태가 거부 됨 상태입니다.   어떻게 해야 합니까?

**답변** **업데이트** 를 클릭 하 여 법적 프로필 정보를 업데이트 해야 합니다. 계속 하기 전에 다음 문서를 참조 하세요.

- [계정 정보 확인](verification-responses.md#what-is-verified-and-how-to-respond)

- [조직 프로필 확인](update-your-partner-profile.md)

문제를 해결할 수 없는 경우 CSP 계정으로 로그인 하 여 파트너 센터 내에서 지원 티켓을 발생 시킵니다.

- 비즈니스 확인 문제의 경우 [지원 요청](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=52ac28f3-d58f-99d9-9846-3df5a6477c54)을 만듭니다.  
- 고용 확인 문제는 [지원 요청](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=c34a5c81-a111-476d-11a4-81c808c37a6b)을 만듭니다.

#### <a name="i-enrolled-in-mpn-program--csp-indirect-reseller-program-but-my-legal-business-profile-verification-status-seems-to-be-stuck-in-the-pending-state-what-should-i-do"></a>MPN program/CSP 간접 재판매인 프로그램에 등록 되어 있지만 합법적인 비즈니스 프로필 확인 상태가 보류 중 상태로 유지 되는 것 같습니다.   어떻게 해야 합니까?

**답변**  CSP 간접 재판매인 법적 업무 프로필이 5 일 이상 후에도 **보류 중인 정규화** 상태에 있는 경우 파트너 센터 [지원](https://partner.microsoft.com/dashboard/support/servicerequests/create?stage=2&topicid=345795c0-26bd-dd95-d291-b78ed4a8edce)을 클릭 하 여 파트너 센터 내에서 지원 티켓을 제기 합니다.

## <a name="questions-about-working-in-partner-center"></a>파트너 센터에서 작업 하는 방법에 대 한 질문

#### <a name="i-dont-know-who-the-partner-center-admin-is-who-can-accept-mpa"></a>파트너 센터 관리자가 MPA를 수락할 수 있는 사람을 알 수 없나요?

**답변** 회사의 전역 관리자는 MPA를 수락 해야 합니다. 회사에서 전역 관리자 역할이 할당 된 사용자를 확인 하려면 **사용자 관리로 이동 합니다.** 검색 상자에 전역 관리자를 입력 합니다. 전역 관리자 역할이 할당 된 사용자의 이름을 가져옵니다. 이러한 사람이 MPA를 수락할 수 있습니다. 회사에 전역 관리자가 없으면 [회사에서 전역 관리자가 필요한 경우 수행할](no-mpn-admin.md)작업을 확인 합니다.

:::image type="content" source="images/user-management-clean.png" alt-text="전역 관리자를 찾는 방법":::


#### <a name="my-reseller-says-that-in-partner-center-they-dont-see-an-option-to-accept-the-mpa-how-can-i-best-support-them"></a>대리점은 파트너 센터에서 MPA를 수락 하는 옵션이 표시 되지 않는다는 것을 의미 합니다. 이러한 기능을 최대한 활용 하려면 어떻게 해야 하나요?

**답변**  대리점에서 파트너 센터에 로그인 할 수 있지만 MPA를 수락 하는 옵션이 표시 되지 않는 경우:

1. 왼쪽의 CSP 탭에 "개요" 페이지가 표시 되나요? 

:::image type="content" source="images/csp.png" alt-text="파트너 센터 메뉴에서 CSP를 찾을 수 있는 위치입니다.":::

파트너 센터 메뉴의 CSP 아래에 "개요"가 표시 되지 않으면 CSP 프로그램에 아직 등록 하지 않은 것입니다. 재판매인은 [csp 프로그램의 간접 공급자 작업에](indirect-reseller-tasks-in-partner-center.md)설명 된 대로 Csp 간접 재판매인으로 등록 하는 프로세스를 시작 해야 합니다. 등록을 완료 한 후에는 MPA를 수락할 수 있습니다.

2. 개요가 표시 되 면 **설정**  ->  **계정 설정**  ->  **법률**  ->  **파트너** 또는 **재판매인** 탭으로 이동 하 여 프로그램 정보로 스크롤합니다. 간접 재판매인 상태 옆에 표시 되는 상태는 무엇 인가요?


상태가 다른 것 이지만 **활성** 상태인 경우에는 법적 비즈니스 프로필 확인이 아직 진행 중이거나 거부 되었음을 의미 합니다. 동일한 페이지에서 재판매인은 확인 상태를 확인 하 고 **자세한 정보** 를 클릭 하 여 문제를 해결 해야 합니다 .이 문제를 해결 하려면 재판매인은 확인을 진행 하는 데 필요한 설명서를 제공할 수 있습니다.

:::image type="content" source="images/active-status.png" alt-text="상태가 활성입니다.":::

재판매인이 법적 비즈니스 프로필 확인을 통과 하도록 지원 해야 하는 경우 다음 옵션을 선택 하 여 티켓을 열도록 요청 합니다.

- 파트너 센터 메뉴에서 **지원** 을 선택한 다음 **일반** > 계정 확인 >을 선택 하 고에 대 한 도움이 필요한 단계를 선택 합니다.


#### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-accept-mpa"></a>파트너 센터에서 MPA를 수락 하는 데 적합 한 역할이 있는지 확인 어떻게 할까요??

**답변** 전역 관리자만 MPA를 수락할 수 있습니다. 파트너 센터에서 역할을 확인 하려면 다음을 수행 합니다.

1. 파트너 센터에서 CSP 간접 재판매인 계정에 로그인 하 고 먼저 CSP 간접 재판매인 계정에 등록 했는지 확인 합니다. 

2. **사용자 관리** 로 이동 하 여 로그인 한 계정에 "전역 관리자" 역할이 할당 되어 있는지 확인 합니다. 그렇지 않으면 MPA를 허용 하는 데 사용할 수 없습니다. 사용자 관리 페이지의 검색 상자에서 전역 관리자를 검색 하 고 사용자에 게 MPA를 수락 하도록 요청 합니다.

:::image type="content" source="images/user-management-clean.png" alt-text="파트너 센터에서 사용자의 역할을 식별 합니다.":::


#### <a name="how-do-i-check-if-my-company-has-accepted-the-mpa"></a>회사에서 MPA를 수락 했는지 확인 어떻게 할까요??

**답변** CSP 간접 재판매인 계정으로 파트너 센터에 로그인 합니다.
CSP의 관리 에이전트 또는 판매 에이전트의 경우 규약 페이지로 이동 하 여 Microsoft 파트너 계약이 표시 되는지 확인할 수 있습니다. 

:::image type="content" source="images/agreement-new.png" alt-text="계정 설정에서 규약을 찾습니다.":::


전역 관리자 인 경우 CSP 개요 페이지로 이동 합니다. 녹색 배너가 표시 되는 경우 모두 해당 CSP 지역에서 간접 공급자를 사용 하 여 모든 설정 됩니다. 그렇지 않은 경우 **수락 및 계속** 단추가 MPA를 수락 하 고 CSP 계정을 활성화 하는 것을 볼 수 있습니다.

:::image type="content" source="images/banner-boxed.png" alt-text="녹색 배너.":::

#### <a name="i-want-to-accept-mpa-but-i-dont-see-any-place-button-to-accept-terms-to-do-so"></a>MPA를 수락 하려고 하지만 어떤 장소 (약관에 동의 하는 단추)가 표시 되지 않나요?

**답변** CSP 간접 재판매인 계정 자격 증명을 사용 하 여 파트너 센터에 로그인 합니다. CSP 개요 페이지로 이동합니다.  **수락 및 계속** 단추가 표시 되지 않고, mpa가 수락 되었음을 알리는 녹색 배너가 없으면 mpa를 수락 하는 데 적합 한 역할이 없음을 의미 합니다. MPA를 수락 하려면 전역 관리자에 게 문의 해야 합니다.

:::image type="content" source="images/agreement-two.png" alt-text="계약을 찾습니다.":::
