# <a name="technical-support-for-on-premises-or-cloud-products"></a>온-프레미스 또는 클라우드 제품에 대 한 기술 지원

**적절 한 역할**:
- 연결 된 전역 관리자 또는 MPN Partner Admin을 MPN 하 여 기술 지원 혜택을 활성화 하세요.
- 활성화 후 액세스 ID 및 계약 ID를 볼 수 있는 MPN 연결 된 사용자 계정
- 클라우드 제품에 대 한 고객을 대신 하 여 지원을 만들 수 있는 관리 에이전트입니다.
- 사용자를 대신 하 여 온-프레미스에서 온-프레미스를 만드는 모든 계정 AAD 또는 MSA.

## <a name="overview"></a>개요

[기술 혜택](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical) 페이지에는 두 가지 별도의 이점에 대 한 정보가 나와 있습니다.

1. **기술 제품 지원 인시던트에 대 한 자세한 내용은 아래를 참조 하세요.**
2. TPD (Technical 예약 판매 and Deployment services)- [추가 정보](technical-benefits.md)

이 문서는 기술 혜택을 활성화 하 고, 액세스 ID 및 계약 ID를 확인 하 고, 지원 되는 제품 및 버전을 확인 하 고, MPN 파트너 혜택을 사용 하 여 기술 지원 인시던트/티켓을 만드는 프로세스를 시작 하는 방법을 알고 있어야 하는 Microsoft 파트너 네트워크 (MPN)의 멤버를 위한 것입니다.

### <a name="steps"></a>단계

1. 기술 혜택을 활성화 합니다 (Global Admin 또는 MPN Partner Admin 필요). 정품 인증 후에는 모든 MPN 사용자 계정이 액세스 ID 및 계약 ID를 확인 하 여 혜택을 계정에 연결할 수 있습니다.

2. 기술 지원 권리 및 지원 되는 제품을 확인 하 여 원하는 제품 및 버전에 MPN 혜택을 사용할 수 있는지 확인 합니다.

3. Microsoft 기술 지원 (최신 온-프레미스 제품): 온-프레미스 제품 지원에 대 한 비즈니스 지원을 사용 합니다.

4. 서명 클라우드 지원 (클라우드 제품): 관리 에이전트 역할이 있는 사용자는 요청을 만들기 위해 **고객 및 서비스 관리** 를 선택 합니다. 각 단계에 대 한 다음 세부 정보를 참조 하세요.

## <a name="view-and-activate-your-technical-support-benefits"></a>기술 지원 혜택 보기 및 활성화

파트너 센터에서 기술 혜택을 보거나 활성화 하는 방법:

1. MPN 연결 계정으로 파트너 센터 [대시보드](https://partner.microsoft.com/dashboard)에 로그인합니다.

   > [!NOTE]
   > 계정이 CSP에만 연결 되 고 MPN에 연결 되지 않은 경우 "이 작업에 대 한 권한이 없습니다." 라는 오류 메시지가 표시 됩니다. MPN 연결 계정이 있어야 합니다.

2. 왼쪽 탐색 메뉴에서 **혜택** 을 확장 하 고 [**기술 혜택**](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical)을 선택 합니다. 기술 혜택 페이지에 조직 활성 골드, 실버 또는 Microsoft Action Pack 멤버 자격 제품이 표시 됩니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-1.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-1.png" alt-text="그러면 기술 혜택을 확장 합니다." :::

   > [!NOTE]
   > 활성 멤버 자격이 없는 경우 "아직 혜택이 없습니다." 라는 메시지가 표시 됩니다. 회사에 활성 회원 자격이 있지만이를 볼 수 없는 경우 잘못 된 계정을 사용 하 여 로그인 한 것입니다. 계정을 확인 하려면 페이지의 오른쪽 위에서 **계정** 아이콘을 선택 하 고 **다른 계정으로 로그인** (필요한 경우)을 선택 합니다.

3. Microsoft 기술 지원 또는 서명 제품 지원을 사용 하려면 혜택을 활성화 해야 합니다. 혜택을 활성화 하려면 **활성화** 단추를 선택 합니다. 활성화에 사용 되는 사용자 계정에는 전역 관리자 또는 MPN Partner Admin 역할이 필요 합니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-2.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-2.png" alt-text="기술 이점 활성화를 선택 합니다." :::

4. 액세스 ID 및 계약 ID는 성공적으로 활성화 된 후 모든 사용자에 게 표시 됩니다. 액세스 ID 및 계약 ID가 표시 되는 경우 Microsoft 기술 지원 및 서명 제품 지원 (골드 및 실버 구성원)을 사용할 수 있습니다.

   활성화 단추가 회색으로 표시 되 면 계정에 전역 관리자 또는 MPN Partner Admin 보안 역할이 없습니다. [사용자 관리](https://partner.microsoft.com/dashboard/account/v3/usermanagement#users) 를 사용 하 고 전역 관리자 또는 MPN 파트너 관리자에 게 필터를 사용 하 여 사용자의 혜택을 활성화 하는 사용자를 찾습니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-3.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-3.png" alt-text="활성화 된 기술 혜택은 액세스 ID 및 계약 ID를 표시 합니다." :::

   > [!NOTE]
   > 각 계정에 Microsoft 기술 지원 또는 서명 제품 지원을 처음 사용할 때 액세스 ID 및 계약 ID가 필요 합니다. 사용자의 계정에 혜택이 연결 되 면 액세스 ID 및 계약 ID를 다시 사용할 필요가 없습니다.

기술 지원은 특정 문제, 오류 메시지 또는 예상 대로 작동 하지 않는 기능에 대 한 문제 해결을 제공 합니다. MPN 제품 지원 및 서명 클라우드 지원 지원은 특정 Microsoft 제품 및 버전에 사용할 수 있습니다 (일부 제품 및 버전은 지원 되지 않음).

> [!NOTE]
> 기술 지원 혜택을 사용 하려면 먼저 활성화 해야 합니다.

Microsoft 파트너 네트워크 멤버 자격의 일부로 파트너는 다음과 같은 기술 지원을 받을 수 있습니다.

## <a name="microsoft-product-support-on-premises"></a>Microsoft 기술 지원 (온-프레미스)

최신 온-프레미스 제품에 대 한 기술 지원을 위해 **Microsoft 기술 지원을** 사용할 수 있습니다. 최근 온-프레미스 제품에는 현재 제품 (N)과 이전 제품 버전 (N-1)만 포함 됩니다. 이전 제품 버전은 [일반 지원](/lifecycle/policies/fixed)내에 있는 경우에만 지원 됩니다. [N-1](https://aka.ms/n-1) 지원 정책을 기반으로 합니다. 지원 되지 않는 제품 및 버전의 경우 관련 유료 지원 옵션을 사용 하거나 파트너 지원 계획을 구매 하세요.

혜택을 처음 사용할 때 액세스 ID 및 계약 ID가 필요 합니다. 혜택은 만료 될 때까지 사용자 계정에 연결 되거나 더 이상 지원 인시던트가 남아 있지 않습니다.

액세스 ID 및 계약 ID가 있는 경우 모든 사용자 계정 (AAD 또는 MSA)에서 온-프레미스 제품에 대 한 기술 지원을 사용할 수 있습니다.

온-프레미스 제품에 대 한 지원 요청을 만들려면 [비즈니스 지원](https://support.serviceshub.microsoft.com/supportforbusiness/create) 을 선택 합니다.

### <a name="how-many-partner-support-incidents-are-remaining"></a>남은 파트너 지원 인시던트 수

이 정보는 비즈니스 온-프레미스 지원 포털에 대 한 지원 에서만 가져올 수 있습니다. 아쉽게도 클라우드 포털 (예: Azure, LCS 및 기타)은이 정보를 표시 하지 않습니다.

비즈니스에 대 한 지원을 사용 하 여 온-프레미스 지원 인시던트를 만드는 프로세스를 시작 해야 합니다. 프로세스를 완료 하지 않으므로 지원 요청을 만들지 않습니다. 필요한 정보는 2 단계에서 표시 됩니다.

1. [비즈니스 지원](https://support.serviceshub.microsoft.com/supportforbusiness/create) 을 열고 windows 10을 선택 하 고 더미 정보를 입력 합니다.

2. 지원 계약이 계정에 이미 연결 되어 있는 경우 지원 인시던트가 표시 됩니다. 온-프레미스 지원의 경우 시작 날짜, 종료 날짜 및 남아 있는 인시던트를 비롯 하 여 더 이상 유효 하지 않은 파트너 지원 계약이 표시 되어야 합니다.

지원 계약에 연결 해야 하는 경우 "지원 플랜을 구입 하거나 추가 해야 합니까?"를 확장 하 고 **계약 추가** 단추를 선택 하 여 액세스 Id 및 계약 ID를 입력 합니다. **사용할 수 없는 지원 계획을** 보 시겠습니까? "를 확장 합니다. 섹션을 참조 하세요. 계약이 만료 되었거나, 남은 인시던트가 없거나, 이전 온-프레미스 제품을 선택한 경우 여기에 표시 되는 파트너 지원도 표시 됩니다 (지원 제품에 대 한 [aka.ms/n-1](https://aka.ms/n-1) 참조).

자세한 내용은 [남은 파트너 지원 인시던트](https://support.microsoft.com/topic/how-many-partner-support-incidents-are-remaining-e8220db5-14f6-93db-4b4a-35766e11b3c3)수를 참조 하세요.

## <a name="signature-cloud-support-cloud-only"></a>서명 클라우드 지원 (클라우드 전용)

골드 또는 실버 역량 파트너는 클라우드 제품에 대 한 기술 지원을 위해 **서명 클라우드 지원을** 사용할 수 있습니다. Action Pack 파트너는 역량을 획득 하지 않는 한 서명 클라우드 지원에 적합 하지 않습니다.  

Azure, Dynamics 365 및 기타 클라우드 제품에 대 한 혜택을 처음 사용 하는 경우 (예외 Microsoft 365) 액세스 ID 및 계약 ID가 필요 합니다. 혜택은 만료 될 때까지 사용자 계정에 연결 되거나 더 이상 지원 인시던트가 남아 있지 않습니다.

> [!NOTE]  
> Azure Portal 사용 하는 경우 지원 계약이 계정에 이미 연결 되어 있는 경우 "액세스 ID가 이미 연결 되어 있습니다 ..." 라는 오류 메시지가 표시 됩니다. 액세스 ID 및 계약 ID를 추가 하는 경우 아래 참조

Azure 요청을 만드는 CSP 파트너는 지원 계약을 추가 하지 않고 "클라우드 솔루션 공급자" 지원 계획을 사용할 수 있습니다.

**관리 에이전트** 역할이 있는 사용자는 왼쪽 탐색 메뉴에서 CSP 아래에 있는 [고객](https://partner.microsoft.com/commerce/customers/list) 을 선택 하 고, **서비스 관리** 를 선택 하 고, **새 요청** 을 선택할 수 있습니다.  관련 작업을 선택 하면 워크 로드 별 포털이 열리고 지원 계약을 추가 하 고 요청을 만듭니다.

### <a name="azure-portal-error-access-id-is-already-associated-"></a>Azure Portal 오류 "액세스 ID가 이미 연결 되어 있습니다 ..."

Azure Portal 사용 하는 경우 도움말 + 지원, 지원 계획, "지원 혜택 링크"를 선택 하 고 액세스 ID 및 계약 ID를 입력 하 여 서명 클라우드 지원을 사용 합니다.

> [!NOTE]
> Microsoft Action Pack에는 서명 클라우드 지원 전용 Microsoft 기술 지원 (온-프레미스)이 포함 되어 있지 않기 때문에 클라우드 지원을 받을 수 없습니다.

오류 메시지가 표시 될 수 있습니다 `This Access ID is already associated to an account. Please contact your Technical Account Manager to obtain a new Access ID` . 이 오류 메시지는 유효 하지 않으므로 무시 해야 합니다. 오류를 사용 하지 않도록 설정 하 고 유용한 진단 오류 메시지로 대체 하도록 요청 했습니다.

`Access ID is already associated …`Azure Portal를 사용 하는 경우 지원 계약을 계정에 두 번 이상 연결 하면 앞의 오류가 표시 됩니다. 아래 시나리오를 참조 하십시오.

1. 골드 또는 실버 기술 혜택이 있고 이전에 온-프레미스 지원 계약을 추가한 경우 Azure Portal를 사용 하 고 두 번째 계약을 동일한 계정에 추가 합니다. 지원 계약은 계정에 한 번만 연결 해야 하므로 Azure에서 지원 혜택을 연결 하지 않아도 됩니다. 계약이 계정에 이미 연결 되어 있을 때 링크 지원 혜택을 사용 하는 경우 오류가 표시 되 고 무시 됩니다. + New 지원 요청를 사용 하 고 지원 인시던트를 정상적으로 만들어 지원 계획을 표시 합니다.

2. 지원 계약이 만료 되었거나 서명 10 팩 (골드 하이브리드 역량) 또는 서명 5 팩 (실버 하이브리드 역량)에 인시던트가 남아 있지 않으면 지원 계약이 잘못 되었기 때문에 숨겨집니다. 지원 혜택을 다시 연결 하면 계약이 이미 연결 되어 있기 때문에 오류가 표시 됩니다. 새 지원 계획을 구매 하 여 Azure 지원 인시던트를 받을 수 있습니다.
