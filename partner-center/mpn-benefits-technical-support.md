# <a name="technical-support-for-on-premises-or-cloud-products"></a>온-프레미스 또는 클라우드 제품에 대한 기술 지원

**적절한 역할:**
- MPN과 연결된 전역 관리자 또는 MPN 파트너 관리자가 기술 지원 혜택을 활성화합니다.
- 활성화 후 액세스 ID 및 계약 ID를 볼 수 있는 MPN 관련 사용자 계정입니다.
- 관리 에이전트를 사용하여 클라우드 제품에 대한 고객 대신 지원을 만듭니다.
- 고객 또는 조직을 대신하여 온-프레미스 지원 요청을 만드는 모든 계정 AAD 또는 MSA입니다.

## <a name="overview"></a>개요

[기술 혜택](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical) 페이지에는 다음과 같은 두 가지 혜택에 대한 정보가 표시됩니다.

1. **기술 제품 지원 인시던트 - 아래 참조**
2. 기술 사전 판매 및 배포 서비스(TPD) - [자세한 정보](technical-benefits.md)

이 문서는 MPN(Microsoft 파트너 네트워크) 구성원을 위한 것으로, 기술 혜택을 활성화하는 방법, 액세스 ID 및 계약 ID 보기, 지원되는 제품 및 버전 및 MPN 파트너 혜택을 사용하여 기술 지원 인시던트/티켓을 만드는 프로세스를 시작하는 방법을 알고 있어야 합니다.

### <a name="steps"></a>단계

1. 기술 혜택을 활성화합니다(전역 관리자 또는 MPN 파트너 관리자 필요). 활성화 후 모든 MPN 사용자 계정은 액세스 ID 및 계약 ID를 보고 해당 계정에 혜택을 연결할 수 있습니다.

2. 기술 지원 자격 및 지원되는 제품을 확인하여 MPN 혜택을 의도한 제품 및 버전에 사용할 수 있는지 확인합니다.

3. Microsoft 제품 지원(최근 온-프레미스 제품용): 온-프레미스 제품 지원에 비즈니스 지원 사용

4. 시그니처 클라우드 지원(클라우드 제품용): 관리 에이전트 역할이 있는 사용자는 **고객 및 서비스 관리를** 선택하여 요청을 만듭니다. 각 단계에 대한 다음 세부 정보를 참조하세요.

## <a name="view-and-activate-your-technical-support-benefits"></a>기술 지원 혜택 보기 및 활성화

파트너 센터 기술 혜택을 보거나 활성화하는 방법:

1. MPN 연결 계정으로 파트너 센터 [대시보드](https://partner.microsoft.com/dashboard)에 로그인합니다.

   > [!NOTE]
   > 계정이 CSP에만 연결되고 MPN에 연결되지 않은 경우 "이 작업에 대한 권한이 없습니다."라는 오류 메시지가 표시됩니다. MPN 연결 계정이 있어야 합니다.

2. 왼쪽 탐색 메뉴에서 **혜택** 을 확장하고 **기술 혜택** 을 선택합니다. 기술 혜택 페이지에는 조직의 활성 골드, 실버 또는 Microsoft Action Pack 멤버 자격 제안이 표시됩니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-1.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-1.png" alt-text="기술 혜택, 혜택 확장" :::

   > [!NOTE]
   > 활성 멤버 자격 제안이 없으면 "아직 혜택이 없는 것 같습니다."라는 메시지가 표시됩니다. 회사에 활성 멤버 자격 제안이 있지만 볼 수 없는 경우 잘못된 계정을 사용하여 로그인한 것입니다. 계정을 확인하려면 페이지 오른쪽 위에 있는 **계정** 아이콘을 선택하고 **다른 계정으로 로그인(필요한 경우)을** 선택합니다.

3. Microsoft 제품 지원 또는 서명 제품 지원을 사용하려면 혜택을 활성화해야 합니다. 혜택을 활성화하려면 **활성화** 단추를 선택합니다. 활성화에 사용되는 사용자 계정에는 전역 관리자 또는 MPN 파트너 관리자 역할이 필요합니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-2.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-2.png" alt-text="기술 혜택 활성화를 선택합니다." :::

4. 활성화에 성공하면 액세스 ID 및 계약 ID가 모든 사용자에게 표시됩니다. 액세스 ID 및 계약 ID가 표시되면 Microsoft 제품 지원 및 서명 제품 지원(골드 및 실버 멤버)을 사용할 수 있습니다.

   활성화 단추가 회색으로 표시되면 계정에 전역 관리자 또는 MPN 파트너 관리자 보안 역할이 없습니다. [사용자 관리를](https://partner.microsoft.com/dashboard/account/v3/usermanagement#users) 사용하고 전역 관리자 또는 MPN 파트너 관리자를 필터링하여 혜택을 활성화할 사용자를 찾습니다.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-3.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-3.png" alt-text="활성화된 기술 혜택에 액세스 ID 및 계약 ID 표시" :::

   > [!NOTE]
   > 각 계정에 대해 Microsoft 제품 지원 또는 서명 제품 지원을 처음 사용할 때 액세스 ID 및 계약 ID가 필요합니다. 혜택이 계정에 연결되면 액세스 ID 및 계약 ID를 다시 사용할 필요가 없습니다.  

## <a name="technical-support-entitlement-and-supported-products"></a>기술 지원 자격 및 지원되는 제품

기술 지원은 특정 문제, 오류 메시지 또는 예상대로 작동하지 않는 기능에 대한 문제 해결을 제공합니다. MPN 제품 지원 및 시그니처 클라우드 지원 지원은 특정 Microsoft 제품 및 버전에 사용할 수 있습니다(모든 제품 및 버전이 지원되지 않음).

> [!NOTE]
> 기술 지원 혜택을 사용하려면 먼저 활성화해야 합니다.

Microsoft 파트너 네트워크 멤버 자격의 일부로 파트너는 다음과 같은 기술 지원을 받을 수 있습니다.

|  기술 지원 |  Action Pack |  실버 역량 | 골드 역량 |
|:------------------:|:------------:|:------------------:|:---------------:|

| 시그니처 클라우드 지원<br>클라우드 제품 전용 | | 클라우드 역량:<br>제한 없음<br><br>기타 역량:<br>5 인시던트 | 클라우드 역량:<br>제한 없음<br>          <br>기타 역량:<br>10개 인시던트   | 

클라우드 역량에는 보안 역량, 클라우드 Business Applications, 클라우드 플랫폼, 클라우드 생산성, Enterprise Mobility Management, 소형 및 중간급 클라우드 솔루션이 포함됩니다.

## <a name="microsoft-product-support-on-premises"></a>Microsoft 제품 지원(온-프레미스)

최신 온-프레미스 제품에 대한 기술 지원에 **Microsoft 제품** 지원을 사용할 수 있습니다. 최근 온-프레미스 제품에는 현재 제품(N) 및 이전 제품 버전(N-1)만 포함됩니다. 이전 제품 버전은 [일반 지원](/lifecycle/policies/fixed)내에 있는 경우에만 지원됩니다. [N-1](https://aka.ms/n-1) 지원 정책을 기반으로 합니다. 지원되지 않는 제품 및 버전의 경우 관련 유료 지원 옵션을 사용하거나 파트너 지원 플랜을 구매합니다.

혜택을 처음 사용할 때 액세스 ID 및 계약 ID가 필요합니다. 혜택은 만료되거나 더 이상 지원 인시던트 남아 있지 않을 때까지 사용자 계정에 연결됩니다.

모든 사용자 계정(AAD 또는 MSA)은 액세스 ID 및 계약 ID가 있는 경우 온-프레미스 제품에 대한 기술 지원을 사용할 수 있습니다.

[비즈니스 지원을](https://support.serviceshub.microsoft.com/supportforbusiness/create) 선택하여 온-프레미스 제품에 대한 지원 요청을 만듭니다.

### <a name="how-many-partner-support-incidents-are-remaining"></a>남은 파트너 지원 인시던트 수

이 정보는 비즈니스 지원 온-프레미스 지원 포털에서만 얻을 수 있습니다. 아쉽게도 클라우드 포털(예: Azure, LCS 등)에는 이 정보가 표시되지 않습니다.

비즈니스 지원을 사용하고 프로세스를 시작하여 온-프레미스 지원 인시던트 만들기를 시작해야 합니다. 프로세스를 완료하지 않으므로 지원 요청을 만들지 않습니다. 필요한 정보는 2단계에 표시됩니다.

1. [비즈니스 지원을](https://support.serviceshub.microsoft.com/supportforbusiness/create) 열고 windows 10을 선택하고 더미 정보를 입력합니다.

2. 지원 계약이 계정에 이미 연결되어 있는 경우 지원 인시던트 표시되어야 합니다. 온-프레미스 지원의 경우 시작 날짜, 종료 날짜 및 남은 인시던트 등 파트너 지원 계약이 더 이상 유효하지 않은 경우 표시되어야 합니다.

지원 계약을 연결해야 하는 경우 "지원 플랜을 구입하거나 추가해야 합니까?"를 확장하고 **계약 추가** 단추를 선택하여 액세스 ID 및 계약 ID를 입력합니다. 사용할 **수 없는 지원 플랜을 확인하시겠습니까?**"를 확장합니다. 파트너 서명 정보를 보려면 섹션을 참조하세요. 계약이 만료되거나, 남은 인시던트 또는 이전 온-프레미스 제품을 선택한 경우에만 여기에 파트너 지원이 표시됩니다(지원 제품에 대한 [aka.ms/n-1](https://aka.ms/n-1) 참조).

자세한 내용은 [남은 파트너 지원 인시던트 수?를 참조하세요.](https://support.microsoft.com/topic/how-many-partner-support-incidents-are-remaining-e8220db5-14f6-93db-4b4a-35766e11b3c3)

## <a name="signature-cloud-support-cloud-only"></a>시그니처 클라우드 지원(클라우드에만 해당)

골드 또는 실버 역량 파트너는 클라우드 제품에 대한 기술 지원에 **시그니처 클라우드 지원** 사용할 수 있습니다. Action Pack 파트너는 역량을 획득하지 않는 한 시그니처 클라우드 지원 자격이 없습니다.  

Azure, Dynamics 365 및 기타 클라우드 제품(예외 Microsoft 365)에 대한 혜택을 처음 사용할 때 액세스 ID 및 계약 ID가 필요합니다. 혜택은 만료되거나 더 이상 지원 인시던트 남아 있지 않을 때까지 사용자 계정에 연결됩니다.

> [!NOTE]  
> 를 사용하는 경우 Azure Portal. 지원 계약이 이미 계정과 연결된 경우 "액세스 ID가 이미 연결되어 있습니다..."라는 오류 메시지가 표시됩니다. 액세스 ID 및 계약 ID를 추가할 때 아래 참조

Azure 요청을 만드는 CSP 파트너는 지원 계약을 추가하지 않고 "클라우드 솔루션 공급자" 지원 계획을 사용할 수 있습니다.

관리 **에이전트** 역할이 있는 사용자는 왼쪽 탐색 메뉴의 CSP 아래에서 [고객을](https://partner.microsoft.com/commerce/customers/list) **선택하고, 서비스 관리** 를 선택하고, 새 **요청을** 선택할 수 있습니다.  관련 워크로드를 선택하면 워크로드별 포털이 열리고 지원 계약을 추가하고 요청을 만듭니다.

### <a name="azure-portal-error-access-id-is-already-associated-"></a>Azure Portal 오류 "액세스 ID가 이미 연결되어 있습니다..."

Azure Portal 사용하는 경우 도움말 + 지원, 지원 계획, "지원 혜택 연결"을 선택하여 시그니처 클라우드 지원 사용하고 액세스 ID 및 계약 ID를 입력합니다.

> [!NOTE]
> Microsoft Action Pack 있는 경우 시그니처 클라우드 지원 포함되지 않으므로 클라우드 지원을 받지 못합니다. Microsoft 제품 지원(온-프레미스)만 포함됩니다.

오류 메시지가 표시 될 수 있습니다 `This Access ID is already associated to an account. Please contact your Technical Account Manager to obtain a new Access ID` 합니다. 이 오류 메시지는 유효하지 않으며 무시해야 합니다. 오류를 사용하지 않도록 요청하고 유용한 진단 오류 메시지로 대체했습니다.

Azure Portal 사용할 때 `Access ID is already associated …` 지원 계약을 계정에 두 번 이상 연결하면 위의 오류가 표시됩니다. 아래 시나리오를 참조하세요.

1. 골드 또는 실버 기술 혜택이 있고 이전에 온-프레미스 지원에 대한 계약을 추가한 경우 Azure Portal 사용하여 동일한 계정에 계약을 두 번째로 추가합니다. 지원 계약을 한 번만 계정에 연결하면 되므로 Azure에서 지원 혜택을 연결할 필요가 없습니다. 계약이 이미 계정에 연결된 경우 링크 지원 혜택을 사용하는 경우 오류가 표시되며 무시해야 합니다. + 새 지원 요청 사용하고 정상적으로 지원 인시던트 만들기 및 지원 계획이 표시됩니다.

2. 지원 계약이 만료되었거나 서명 10 팩(골드 하이브리드 역량) 또는 서명 5 팩(실버 하이브리드 역량)에 남은 인시던트는 없으면 지원 계약이 잘못되어 숨겨집니다. 지원 혜택을 다시 연결하면 계약이 이미 연결되어 있으므로 오류가 표시됩니다. 새 지원 계획을 구입하여 Azure 지원 인시던트도 얻을 수 있습니다.

파트너 유료 지원 제품은 아래의 파트너 지원 플랜을 참조하세요.

## <a name="partner-support-plans"></a>파트너 지원 플랜

기술 지원 및 다양한 추가 서비스 옵션을 포함하는 Microsoft에서 유료 지원을 받을 수 있습니다.

파트너 [지원 계획을 비교하거나 파트너별 플랜에](https://partner.microsoft.com/support/partnersupport) 대해 다음을 확인할 수 있습니다.

- [파트너용 프리미어 지원](https://partner.microsoft.com/support/microsoft-services-premier-support) - Microsoft의 최상위 기술 지원 수준
- [파트너에 대한 고급 지원](https://partner.microsoft.com/support/advanced-cloud-support) - 클라우드 제품만
